4GL_ILLEGAL_DATA

4GL_ILLEGAL_DATA determines how COBOL data that is considered illegal by the database will be converted before it is stored in the database.

The default value is NULL. This causes all illegal data (except key fields) to be converted to null before it is stored.

The value CONVERTED causes the following conversions to occur:

Illegal LOW-VALUES:

minimum possible value
(0 or -99999...)
minimum possible date value (01/01/01)

Illegal HIGH-VALUES:
                                         

maximum possible value (99999...)
maximum possible date value (database dependent)

Illegal SPACES:

zero (or 01/01/01 for a date field)

Illegal data in key fields is always converted, regardless of the value of this configuration variable.

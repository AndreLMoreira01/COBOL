A_MSSQL_DEFAULT_OWNER

In SQL Server, tables are named database.user.tablename. The value of A_MSSQL_DEFAULT_OWNER specifies the name of the user. If this variable is set, it must match the owner of a table. Tables are referenced as owner.table, where owner is the value of A_MSSQL_DEFAULT_OWNER. If this variable is set to a value for which owner.table does not exist, the interface fails to access the table. Note that newly created tables are owned by the useres who create them, not the owner named in A_MSSQL_DEFAULT_OWNER. The default value of this configuration variable is blank, causing the interface to call an internal stored procedure to determine the actual owner of the table. Note that Acucorp discourages multiple tables with the same tablename in a single database.

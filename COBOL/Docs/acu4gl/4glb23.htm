A_MSSQL_FORCED_INDEX

A_MSSQL_FORCED_INDEX causes the Acu4GL product to attempt to use an actual index that matches the key used with the table. In order for this variable to have any effect, the index matching the key must be a unique index.

While the default value is "0" (off, false, no), this configuration variable can also take values of "On" (true, yes). If you don't use the forced-index feature, the Acu4GL product adds an ORDER BY clause to the SELECT, which can add to the processing time.

Acucorp has recently discovered that SQL Server does not guarantee that an index will be used if this variable is set to 1, and so the order of records returned to the COBOL program is not guaranteed to be correct in that case. Because of this, we highly discourage setting this variable "On".

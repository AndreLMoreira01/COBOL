6.3.1 Components of the Licensing Mechanism

As described in section 1.3.1 of this document, a license file is created from the product code and product key by the Activator utility.

The Product Code and Product Key

In order to obtain the product code and product key pair for your end user, you need to provide Acucorp with the following information:

Host operating system:

Acucorp needs information about the target machine, such as the name of the operating system (OS), the release version of the OS, and the type of processor (such as Intel or Sparc). For example:

Windows XP Professional (Intel)

IBM AIX 5L 5.2 32-bit

HP-UX 11.0 RISC 64-bit

SCO UNIX Open Server 5.0.6

DG/UX SVR4 R4.11 (Intel)

Linux (glibc 2.2) (Intel)

Solaris 8.0 (SPARC) 64-bit

Number of users:

You also need to specify how many users will be accessing the application at the same time (concurrent users). For example, if a site needs access to an application from a number of machines, but only by one user at a time, this would require a different license file than a site where multiple users access the same application concurrently.

Acucorp products:

You need to specify which Acucorp products will be run with your application. Remember to specify the version number of the product using three digits, such as 6.2.0.

Once your Acucorp representative has this information, the product code and product key pair(s) will be shipped in accordance with your License Agreement.

The Activator Utility

In addition to the product code and product key, you will be given the following files, based on the target platform for the license file:

Target Platform Required Files

Windows

activator.exe

UNIX/Linux

activator

acushare*

VMS

activator.exe

HP e3000 with MPE/iX

activator

*After all necessary license files are installed and the applications are ready to run, UNIX and Linux platforms use a utility called acushare to act as the license manager. The acushare utility must be distributed with all UNIX/Linux products. For more information on acushare, see section 8.2, "License File Management Under UNIX/Linux."

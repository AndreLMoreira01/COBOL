2.2 Installation on Windows Platforms

Windows installation requires little interaction; the setup program from Acucorp copies the files into a directory you designate, or "C:\Acucorp\Acucblxx" by default.

After the installation is complete, if you have installed both the compiler and runtime, you can begin to compile and execute your COBOL programs. Basic compilation and execution techniques are described in section 2.3 and section 2.4.

Note: If you move or delete any ".dll" files that have been installed, your Acucorp products may not run.

You will need to have your product codes and product keys and the Acucorp product media available to perform this operation. Your Acucorp products may be delivered on one CD-ROM or on several diskettes. Refer to the appropriate installation instructions below.

Note: If you are installing a Windows runtime ("wrun32.exe") and an Alternate Terminal Manager (ATM) runtime ("run32.exe") on the same machine, be aware that the Activator utility creates a license file named "wrun32.alc" for each of them. You must use one of the following precautions:

  • Install the ATM runtime and rename the license file to match the executable (change "wrun32.alc" to "run32.alc") before installing the Windows runtime.
  • Or make a back-up copy of the Windows runtime license file prior to installing (and renaming) the ATM runtime.

This will avoid a situation where the Activator overwrites the license file for the second runtime.

AcuBench and AcuODBC® must be installed locally (on the client). Server-side products such as AcuServer and AcuConnect® are installed only on server machines.

4.4 Error Messages

SQL error messages reported during program execution are directed to standard error output. Refer to your SQL reference manual for the meanings of SQL error messages. Programmers can check for error codes in their programs by examining the values of SQLSTATE and SQLCODE.

SQL Server Users

You can map message numbers from SQL Server to SQLSTATE values. Follow these steps:

  1. Create a file with the following format:

    2. nnnn XXXXX message string

    where:

    nnnn is the message number returned from SQL Server.

    XXXXX is a five-character value that is returned via SQLSTATE.

    The message string is an optional value that is returned via SQLERRMC. If message string is not set, it defaults to the message string returned from SQL Server. Note that the message string from SQL Server is usually longer than the 70 characters available in SQLERRMC, so using that default can result in truncated messages.

  2. Set the configuration variable ASQL_ERROR_MAP_FILE to point to the file you have created.

    3. The messages are now mapped when errors occur.

MySQL Users

Note that some MySQL drivers do not correctly clear their error buffers after reporting an error. This can cause the driver to produce an infinite loop reporting the same error message. If you keep receiving the same error message over and over, this could be the source of the problem. Please check with your driver manufacturer for detail.

DB2 Users

For error messages generated by DB2 Connect or other connectivity software, refer to your product documentation.

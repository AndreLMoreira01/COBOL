ASQL_DISCONNECT_ACTION

This variable determines what happens if a transaction is in progress when a connection is closed for any reason. Valid values are COMMIT and ROLLBACK.

COMMIT

Any transaction that was started by the interface will be committed before the connection is closed.

ROLLBACK

Any transaction that was started by the interface will be rolled back before the connection is closed.

Note that the action determined by this variable will happen regardless of how the connection is closed (either explicitly or due to program termination), and regardless of how the transaction was begun (either through other configuration variables or explicitly coded into the COBOL program.) At the time a connection is closed under control of the interface, either a "commit transaction" or a "rollback transaction" is issued to the server, until the server returns an error about no transaction being in progress.

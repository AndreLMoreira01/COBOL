2.6.1 Web Service Provider

To provide COBOL Web services, you componentize your ACUCOBOL-GT application using the Transoft Development Suite as described in section 2.2.3.1, then you generate Web services proxies for the client. Rather than selecting Javabean for the client proxy as described in step 4, you select J2EE Web services (generic) or (individual). "Generic" causes Java stub proxy classes to be generated generically for all services, and "Individual" causes Java stub proxy classes to be generated individually for each service.

These proxy classes can be used directly by any J2EE container in the Java environment for onward delivery of the Web service(s). The proxies have been designed to reveal your service components using the protocols and standards that are required by a J2EE-based Web services framework.

Complete the procedure for componentizing your application as described in section 2.2.3.1. When done, publish details of the services in one of the UDDI repositories provided by IBM, Sun, Microsoft, or others. Alternatively, you can generate SOAP for Java (SOAP4J) proxy classes that can be used natively by any Java client application without the need for WSDL.

2.2.2.6 Running the Java interoperability sample programs

Your ACUCOBOL-GT distribution includes Java interoperability sample programs. You will find them in the \acugt\sample\java\ directory where ACUCOBOL-GT is installed. Following are instructions for setting up and running the samples.

  1. Install the JDK Version 1.4.2 or greater on your machine. A JDK is required because the Java compiler must build the samples.
  2. Add "CVM.jar" and the current directory to the CLASSPATH environment variable. (This is used for Java calling COBOL.) The "CVM.jar" file is in the installation \bin directory. For instance, your entry should look like this:
    3. CLASSPATH=C:\Acucorp\Acucbl700\AcuGT\bin\CVM.jar;.;
  3. Edit the file called "config" and use it as a configuration file for running the samples.
    1. For COBOL calling Java, use the JAVA_LIBRARY_NAME variable to specify the Java library to call ("jvm.dll" or "libjvm.so"). On Windows, your entry might look like this:
      2. JAVA_LIBRARY_NAME=jvm.dll

      You can add a fully qualified path to the name if necessary.

    2. Add the options variables, JAVA_OPTIONS and PRELOAD_JAVA_LIBRARY. For Java calling COBOL, JAVA_OPTIONS should specify the same CLASSPATH as the environment like this:
      3. JAVA_OPTIONS=-Djava.class.path=C:\Acucorp\Acucbl700\AcuGT\bin\CVM.jar;.;

      The variable PRELOAD_JAVA_LIBRARY=1 tells the runtime to load the JVM upon initialization.

  4. Add the location of "jvm.dll" or "libjvm.so" to the PATH environment variable. On Windows, the entry should look like: 
    5. PATH=<additional path info>;C:\Program Files\Java\jdk1.5.0_03\jre\bin\client;

    For convenience, it may be helpful to have the Java \bin directory in the PATH as well. On Windows, that directory might be C:\Program Files\Java\jdk1.5.0_03\bin.

  5. Add the location of the runtime DLLs and shared libraries to the variable LD_LIBRARY_PATH. This entry should look like this:
    6. LD_LIBRARY_PATH=C:\Acucorp\Acucbl700\AcuGT\bin
  6. Compile the Java program "JavaCallingCobol.java" with the command:
    7. javac JavaCallingCobol.java
  7. Compile "CobolCallingJava.java" with:
    8. javac CobolCallingJava.java
  8. Ensure that COPYPATH includes the \acugt\sample\ directory. The program "CobolCallingJava.cbl" requires the copyfile "java.def."
  9. Compile the COBOL program "JavaCallingCobol.cbl" with the command: 
    10. ccbl/ccbl32 -x -Ga JavaCallingCobol.cbl
  10. Compile "CobolCallingJava.cbl" with: 
    11. ccbl/ccbl32 -x -Ga CobolCallingJava.cbl
  11. Run the Java calling COBOL test:
    12. java -Djava.library.path=<directory runcbl/wrun32 located> JavaCallingCobol cvm
  12. Check the console log output to ensure that the program finished successfully.
  13. Run the COBOL calling Java test:
    14. runcbl CobolCallingJava 
or 
wrun32 CobolCallingJava
  14. Check the log file "CVM.log" to ensure that the program finished successfully.

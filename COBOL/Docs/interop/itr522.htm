5.5 Calling .NET from COBOL

You can use .NET assemblies in your COBOL programs in one of two ways. You can:

  • Use ACUCOBOL-GT's .NET bridging interface, "AcuToNet.dll". This interface gives you access to .NET assemblies, both graphical and non-graphical, including user controls and Windows forms controls. You do not need to know object-oriented programming or the NET Framework.

    • This interface includes a graphical utility known as the .NET Definitions Generator, or "NETDEFGEN.exe". This utility allows you to view and select .NET assemblies for translation to a COBOL COPY file. COBOL programmers already familiar with the DISPLAY, CREATE, MODIFY, INQUIRE, and DESTROY statements can immediately write COBOL programs that consume .NET assemblies.

  • Use a partner's component adapter technology. The component adapter method exposes your COBOL service components to the .NET platform in C# so you can view and consume the services in Visual Studio .NET. It requires that you define your COBOL program as a service and use the component adapter broker or listener to provide access to that service.

    • If you choose to implement component adapter technology, your applications are poised for use as Web services, and they can be quickly deployed on other client platforms. The component adapter's service-oriented architecture provides efficient scaling when large numbers of clients are involved.

7.7 COBOL Web Services

Using component adapter technology, a Transoft technology available through Acucorp, you can expose your COBOL applications as Web services for use in Web services deployments. With this add-on, you can transform existing COBOL business rules into reusable service components and then expose them as one of three types of Web services:

  • Native Web services - Your COBOL services are wrapped in Web Service Description Language (WSDL) or SOAP for Java so they can run natively in a Web services environment.
  • .NET Web services - Your COBOL services are wrapped in C# so that your services can run in a Microsoft .NET environment.
  • J2EE Web services - Your COBOL services are wrapped in Java so they can run in a J2EE application server environment.

To expose your COBOL code as services, you use a special integrated development environment (IDE) to name the service, describe the service inputs and outputs, and provide other essential data. You then group services into components and generate special code for communicating with a service broker. (A broker is responsible for managing client requests and launching the COBOL services.) Using the IDE, you also generate Web service proxy stubs, which are essentially subroutines in WSDL, C#, or Java that receive and translate calls for the COBOL Web service, temporarily standing in for the service itself.

When deployed, the service broker listens on a configured TCP port for service requests, which are sent via standard interfaces. The broker uses a service description repository to determine the appropriate service to launch. The service is activated, and the broker returns a response to the calling client.

Refer to Chapters 2 and 5 of this guide for more information on deploying COBOL Web services in Java and .NET environments.

In addition, ACUCOBOL-GT supports Web services via .NET client-side control interfaces. This means that an ACUCOBOL-GT program can consume or invoke a Web service via a .NET control. Also, a server-side Web service can consume or invoke an ACUCOBOL-GT server-side program. Refer to section 5.6 for more information.

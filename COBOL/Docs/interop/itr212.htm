2.2.3 Using Component Adapter Technology

Component adapter technology, a Transoft technology available through Acucorp, adapts application components that were written in COBOL for alternate environments, such as Java. The component adapter technology exposes your COBOL service components to the Java platform as Javabeans so you can view and consume the services in a Java development environment such as JBuilder(TM), WebLogic Workshop(TM), or WebSphere® Studio.

To provide ACUCOBOL-GT services to a Java program, you use Transoft's integrated development environment (IDE) to identify the COBOL functions that you want to expose, and then you generate Javabean proxies for those functions. Proxies are stubs in the client's native language that present the service component to the consumer application in a familiar way.

A broker on the COBOL server interprets and routes requests for the COBOL component. It listens for requests sent through the Javabean proxy, then uses a service description repository (SDR) to determine the appropriate COBOL component to launch. It also provides context management options and additional configuration control for the site manager, as well as thread safety for the COBOL components.

Component adapters can be run under J2EE application servers such as BEA WebLogic Server(TM) and IBM® WebSphere, too. In this case, your COBOL application becomes, in effect, a Javabean component that is managed by the application server along with other Javabean components. The COBOL broker waits for requests from the application server, and launches COBOL components on demand. (Refer to section 2.5 for more information on using ACUCOBOL-GT applications with J2EE application servers.)

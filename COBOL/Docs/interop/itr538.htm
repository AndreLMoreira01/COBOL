5.5.3 Using Component Adapter Technology

As an alternative to using ACUCOBOL-GT's .NET interface, you can use component adapter technology to integrate your ACUCOBOL-GT program with a .NET assembly or service. Component adapters are a Transoft technology, available through Acucorp. They are designed to adapt application components that were written in COBOL, or any language, for other environments. The advantage of component adapters is that you can expose your COBOL application to a variety of client technologies-.NET, Java, COM, Web services-from the same COBOL source, and the service-oriented architecture scales extremely well.

You can use component adapter technology to do two main things:

  1. Expose COBOL programs to .NET. Using a COBOL language component adapter, you expose existing COBOL business rules as .NET objects so that applications operating in a .NET Framework can gain access to yours. First, you use a development tool to name the service, describe the service inputs and outputs, and so on. You then group services into components and generate an ACUCOBOL-GT skeleton program to represent each of these components. You then insert your own new or existing COBOL code into the skeleton program.

    2. You also generate a connector for your COBOL components. For .NET, you generate a C# proxy stub that presents the COBOL component in the language native to Visual Studio .NET. With a C# proxy, a Windows developer can incorporate the COBOL component into the .NET application. Note that proxies can also be generated for Javabean, SOAP, or COM clients as well as others.

    A component broker manages requests for the COBOL component. The broker uses a service description repository (SDR) to determine the appropriate component to launch.

  2. Interact with .NET Web services. ACUCOBOL-GT programs can interact with .NET Web services using the same facilities described above.

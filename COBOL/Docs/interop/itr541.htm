5.5.3.3 Deployment

To deploy your COBOL services in a .NET environment, do the following:

COBOL server

  1. Upload the COBOL application component and ".sdl" files to the production server. Note that although the .NET assembly is running on a Windows machine, the COBOL service can be running on a Windows or UNIX server.
  2. Install and configure the ACUCOBOL-GT runtime.
  3. Install and configure the component broker. On UNIX, configure the component broker using a configuration file. On Windows:
    1. Launch the Component Broker Administrator. The administrator Property sheet appears.
    2. Click the Settings tab and provide configuration information as desired.
  4. Start the component broker. On UNIX, start the broker (a daemon) via the command line. On Windows, click Start on the General tab of the administrator software. Select Automatic Startup if you want to start the component broker (a Windows service) automatically each time you start up your PC.

Windows server

  1. Install the .NET application, incorporating the C# proxies that were generated for the .NET client environment.
  2. Begin the client process.

2.6.2 Web Service Consumer

Consumers of the Web service use their own tools to incorporate the client proxy into their application. On the WebLogic platform, for instance, they use the Web Services Tool Kit (WSTK) of WebLogic Workshop to update their Java application.

As with any Web service, the consumer application is responsible for interrogating the UDDI repository to obtain a WSDL document describing the service. If SOAP4J proxies were provided, you can include them directly in your Java application and avoid having to interrogate the UDDI.

The application must also contain logic for connecting to the service provider's server based on information contained in the WSDL document or SOAP4J proxy. It must also be programmed to issue a Web service request to the specified URL via XML/SOAP.

When the Transoft component broker receives the request, it determines which service has been requested and invokes the service with the supplied input parameters. The results are passed back to the component broker, which parcels them into a SOAP message and delivers the message to the consumer application. The consumer application must be programmed to handle the SOAP package that is obtained. The COBOL Web service becomes a standard part of the J2EE Web services environment.

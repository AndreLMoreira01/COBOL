2.5.1 Working with J2EE Application Server Products

ACUCOBOL-GT applications can be integrated with application server platforms using any of the methods described in this chapter. Your Java components are managed by the application server, but they can still gain access to COBOL using Acucorp's CVM class, component adapter technology, the C$SOCKET routine, CGI programs, the runtime DLL, or named pipes.

Similarly, your COBOL applications can access your Java components, managed by the application server, via Acucorp's C$JAVA routine, component adapter technology, the C$SOCKET routine, the CALL statement, the C$SYSEM routine, or named pipes.

Note that the use of component adapter technology involves two component brokers: the application server broker and a COBOL broker.

The component broker on the COBOL server interprets and routes requests for the COBOL component, in much the same way that WebLogic or WebSphere interprets and routes requests for J2EE components. The COBOL broker listens for requests sent through the Javabean connector via the Java application server, then uses an SDR to determine the appropriate COBOL component to launch. It also provides context management options and additional configuration control for the site manager, as well as thread safety for the COBOL components.

With component adapter technology, your COBOL application becomes, in effect, a Javabean component that can be managed by the application server along with other Javabean components.

Case study

A major water utility provides business on demand to millions of customers every day. It uses a combination of IBM and Acucorp technologies to respond quickly to service requests while minimizing disruption to the surrounding roadways.

The utility company has an IBM mainframe application for scheduling and managing fieldwork. The application has a Java interface, accesses DB2® data, and uses CICS® and WebSphere MQ to manage transactions.

To coordinate projects with local street authorities, the utility also runs a critical UNIX-based application written in ACUCOBOL-GT.

To integrate the processes and provide real-time business on demand, the utility adopted IBM WebSphere Application Server and Transoft's component adapter technology. Today, users can access both the mainframe and ACUCOBOL-GT applications from the same Java user interface. There, they can dynamically schedule fieldwork, coordinate with street authorities, and communicate with employees in real time via mobile and Web browsers.

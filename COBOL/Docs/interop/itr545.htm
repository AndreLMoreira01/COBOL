5.6.2.1 Component adapter method

To provide COBOL Web services, you componentize your ACUCOBOL-GT application using the Transoft Development Suite as described in section 5.5.3, then you generate Web services proxies for the client. Rather than selecting C# for the client proxy as described, you select either Native Web services or .NET Web services.

Choose Native Web services if you want Web Service Description Language (WSDL) proxies to be generated for your COBOL service component. Select .NET Web services if you want C# proxy stubs to be generated.

Note: "Generic" causes proxy classes to be generated generically for all services, and "Individual" cause proxy classes to be generated individually for each service.

Either of these proxy classes can be used directly by any .NET service for onward delivery of the COBOL Web service(s). The proxies have been designed to reveal your service components using the protocols and standards that are required by the .NET Web services framework.

Complete the procedure for componentizing your application as described in section 5.5.3. When done, publish details of the services in one of the UDDI repositories provided by IBM, Sun, Microsoft, or others.

.NET consumers of the COBOL Web service use their own tools to incorporate the Web service proxy into their application, typically Visual Studio .NET.

As with any Web service, the consumer application is responsible for interrogating the UDDI repository to obtain a WSDL document describing the service.

The application must also contain logic for connecting to the service provider's server based on information contained in the WSDL document. It must be programmed to issue a Web service request to the specified URL via XML/SOAP.

When the Transoft component broker receives the request, it determines which service has been requested and invokes the service with the supplied input parameters. The results are passed back to the component broker, which parcels them into a SOAP message and delivers the message to the consumer application.

The consumer application must be programmed to handle the SOAP package that is obtained. The COBOL Web service becomes a standard part of the .NET Web services environment.

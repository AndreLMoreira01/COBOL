3.1 COBOL and Windows

Due to the widespread popularity of the Microsoft® Windows operating system, programmers commonly develop applications for the Windows environment. They perceive that modern applications should have a Windows look and feel complete with floating windows, scroll bars, menus, push buttons-all of the graphical features of a Microsoft application. Further, they want their application to respond to a mouse click or key press. And they want it to run on all of the latest Windows platforms: from Windows 98, ME, and XP to Windows NT, 2000, and 2003.

Fortunately for COBOL developers, ACUCOBOL-GT® is a graphical COBOL language that is uniquely suited for graphical Windows environments. ACUCOBOL-GT not only supports all major Windows technologies, but it runs on most 32-bit versions of the Windows operating system. Since its inception, Acucorp has given COBOL developers the tools they need to create modern, graphical, event-driven Windows applications in COBOL.

In this chapter, you will learn how to call dynamic link libraries (DLLs), access the Windows Application Programming Interface (API), use Visual C++ .NET, and enable Windows features like ".wav" files and print spoolers. You'll also learn how to interoperate with applications and data sources that conform to the Open Database Connectivity (ODBC) standard.

Component Object Model (COM), ActiveX®, and .NET are large enough subjects to warrant their own chapters. To learn how to leverage COM and ActiveX in your COBOL programs, refer to Chapter 4 of this guide. To learn to interact with .NET assemblies, refer to Chapter 5.

For information on graphical user interface (GUI) programming (including concepts such as windows, handles, events, methods, menus, and color mapping), please refer to ACUCOBOL-GT User Interface Programming.

6.4.1.1 Defining Internet pathnames: AcuServer configuration

If you are using AcuODBC on the client to access remote Vision data via AcuServer, you define the location of the data files using the AcuODBC configuration manager on the client machine. It doesn't matter whether you are accessing the data across the Internet or across a LAN or WAN.

To open the configuration manager, set up a Data Source Name as described in the AcuODBC User's Guide section 3.1. When you are finished, click Finish and the AcuODBC Configuration property sheet appears.

On the General tab, use the Data Directory field to define the path of the remote data. The Data Directory field simulates the FILE_PREFIX configuration variable for AcuServer, so the remote name notation is the same as described in the AcuServer section of this chapter. For Internet configurations, you include the domain name of the server users are accessing over the Internet. For example, if the name of the Internet server is "condor," you could define the Data Directory field as follows: 

@condor.XYZCorp.com:/usr/data

Alternatively, you could enter the explicit IP address of the server in the Data Directory field, as in the following example: 

@128.110.121.42:/usr/data

Note that if the data resides in the same directory as the XFDs, you can use the XFD Directory field to specify the remote Internet pathname, and leave the Data Directory field blank. To define the AcuServer port number, click the AcuServer tab and complete the Port Number field, 6523 by default.

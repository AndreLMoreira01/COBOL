6.4.1.2 Defining Internet pathnames: AcuODBC Server configuration

In an AcuODBC Server configuration, where data is both accessed and processed remotely, you define the location of the server using the AcuODBC configuration manager on the client machine, then you define the data pathname when you set up the DSN on the server.

On the client machine, you define only the Data Source Name field on the General tab of the AcuODBC Configuration property sheet. Then on the AcuODBC Server tab, you click the Remote SQL Processing button, and enter the name or IP address of the remote AcuODBC Server host. For Internet configurations, you might enter the following Host Name: 

condor.XYZCorp.com

or IP Address: 

128.110.121.42

Then on the server machine, you use the Data Source Configuration Utility to define the name and location of the data source as described in section 9.3 of the AcuODBC User's Guide. There is no need to use remote name notation when defining the XFD and Data Directories because these directories reside on the same machine as AcuODBC Server.

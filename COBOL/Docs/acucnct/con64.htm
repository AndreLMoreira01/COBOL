6.2.1 Server Components Required

The following server components are required for thin client operation:

  • Server operating system: Windows, UNIX, or VMS
  • COBOL application
  • ACUCOBOL-GT® runtime
  • AcuConnect application server
  • AcuAccess file
  • Configuration files (runtime configuration file, server configuration file, server alias file)
  • TCP/IP networking to clients. TCP/IP is not sold or provided by Acucorp.
  • A copy of the AcuConnect license file for thin client connections activated by the Acucorp product installation. This file is named "acurcl.alc".
  • A copy of the runtime license file activated by the Acucorp product installation. This file is named "runcbl.alc" or "wrun32.alc".
  • UNIX servers must run the license manager acushare. The latest version of acushare is included on the AcuConnect distribution media.

Note that you must have a properly functioning network card in your machine in order to install and run AcuConnect.

What is the license file?

AcuConnect is shipped with a product code and product key that defines the licensing capabilities specified in your purchase agreement. When you supply your key information during installation, the Activator creates a license file that contains information such as the product's version number, serial number, expiration date, and server count. After installation, AcuConnect must be able to locate the license file in order to function.

For thin client connections, AcuConnect searches for the license file "acurcl.alc" in the same directory as the acurcl executable. If you move the AcuConnect executable to a new directory, be sure to move a copy of the license file as well. If no license file is found, or if the information in the license file does not permit execution, AcuConnect exits with an error message.

What is acushare?

The acushare utility is required for all UNIX installations that do not have an unlimited license. The acushare utility monitors and enforces the site runtime usage limits.

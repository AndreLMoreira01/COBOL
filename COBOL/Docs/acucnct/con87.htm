8.6 AcuConnect Error Messages

In the course of operations, AcuConnect may produce error messages that inform you of system problems. AcuConnect error messages that appear when acurcl cannot be started are seen on the server by the system administrator who wants to start acurcl or on a client by a user who wants to establish a connection to acurcl and start a client runtime.

Error messages that appear when a remote program cannot be started via acurcl may be different depending on whether AcuConnect is deployed in a distributed processing or thin client environment. Refer to section 8.7.1 and section 8.8.1 for the various messages that are generated for distributed processing and thin client environments, respectively, when a remote application cannot be started.

acurcl start-up errors

Messages generated as a result of the inability to start acurcl are different depending on the operating system. Two conditions can be responsible for the set of messages that appear when acurcl does not start: a missing or expired AcuConnect license file or a missing AcuAccess file.

In a UNIX/Linux environment, the following message may be produced when the license file is missing:

License file `./acurcl.alc' inaccessible.

The following message indicates that the user license is expired:

This license for the AcuConnect product has expired.
Please contact your software vendor for assistance.

For a missing server access file in a UNIX/Linux environment, the AcuConnect log file displays the following:

Access file `/etc/AcuAccess' not found.

From the graphical control panel in Windows NT, the following message may indicate that the AcuConnect license file is missing or expired or that the server access file is missing:

Data area passed to a system call is too small.

This error may also appear in Windows 2000 Server when a license file is missing.

From the graphical control panel in Windows 2000 Server, the following message may indicate a missing or invalid license file or a missing server access file:

Overlapped i/o operation is in progress.

From the command line in Windows servers, the following message is generated as a result of an invalid license file:

License file `acurcl.alc' is inaccessible.

Also on Windows servers, when the AcuAccess file cannot be found, the resulting message is:

Please check messages in the event log.

and the event log shows an entry of "Handle is invalid."

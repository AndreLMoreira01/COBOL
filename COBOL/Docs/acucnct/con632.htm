6.3.1.20 TC_DELAY_ACTIVATE

This variable lets you control whether the thin client delays sending a CMD-ACTIVATE event to the server until after the Windows notification that caused the event is complete. Note that ActiveX events are never delayed, regardless of the setting of this variable. The default value of "1" (on, true, yes) enables this behavior. Setting TC_DELAY_ACTIVATE to "0" (off, false, no) turns off the delay mechanism.

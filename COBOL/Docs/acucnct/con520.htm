5.1.4 Client/Server Character Mapping

In some client/server deployments, the client and server machines may use different character code sets, particularly when one machine is set up for foreign language characters that use values outside the ASCII character set (decimal values 128 and higher). International character handling is similar in the AcuConnect® thin client environment to that in distributed processing. You create a simple text file that contains the mapping of characters that may be different on the client than on the server. You need to include only the character codes that differ between the two machines. For information about the contents and format of this map file, refer to section 2.5, "International Character Handling."

After you create your map file, you need to point to it using a runtime configuration variable. For a thin client environment, this variable is called TC_MAP_FILE. Character mapping is triggered by the presence of the TC_MAP_FILE variable. Refer to section 6.3.1.36, "TC_MAP_FILE," for more information about this configuration variable.

You need to be sure that the configuration file that contains this variable is referenced in the program's alias definition. Information about the thin client alias file can be found in section 6.3.3, "Creating a Server Alias File."

2.6.2 AGS_RECEIVE_BUFFER_SIZE

This variable helps you tune one element of low-level socket communication between a client and server. AGS_RECEIVE_BUFFER_SIZE controls the size of the low-level receive buffer. The variable must be set before the creation of any sockets. The default value is 16384.

Note that because ACUCOBOL-GT does not control this buffer, changes in its value are not displayed in response to a U debugger command to list runtime memory usage. Changes to the default value may have little, if any, noticeable effect. The default value is probably adequate in most situations.

6.5.6.1 Calling client-side DLLs

Calling a DLL on the display host requires special notation. The name of the DLL called on the display host must begin with "@[DISPLAY]:". For example, to call a DLL named "MYLIB" on the client, you would use the following code: 

CALL "@[DISPLAY]:MYLIB.DLL"

The file path for a called DLL is relative to the directory that contains "acuthin.exe". For example, if "acuthin.exe" is located in C:\Acucorp\Acucbl7xx\AcuGT\bin, then the following call: 

CALL "@[DISPLAY]:MYAPP\MYLIB.DLL"

refers to C:\Acucorp\Acucbl7xx\AcuGT\bin\MYAPP\MYLIB.DLL. Note that the CALL statement ignores CODE_PREFIX when it searches for DLLs on the client. CODE_PREFIX is applied only to actions on the application host.

4.2.2.5 ACURCL_PORT

The ACURCL_PORT variable specifies a particular port number to be used for accessing acurcl on the server host machine. This is especially helpful when the server host machine has a security firewall, because firewalls generally allow access only to specific ports. With this variable, the site can ensure that firewall restrictions are satisfied.

The acurcl daemon can work with privileged port numbers (from 0 to 1023), and with non-privileged port numbers (1024 and higher, up to 32767).

The value of this variable is overridden by any port number assigned on the command line (via the "-n" option) when the daemon is started.

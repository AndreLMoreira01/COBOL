8.8.2.6 Screen Section table handling

The compiler produces some internal data items to hold intermediate values when a Screen Section USING, TO, or FROM data item is indexed or reference modified. In the thin client, this behavior is extended so that it uses the same intermediate item when both the FROM and TO items refer to the same data item, consuming slightly less memory. As a result, a program with a Screen Section that stores its values in tables may be able to use the performance benefit of a lower TC_CONTROL_SYNC_LEVEL configuration setting. For more information about this configuration variable, refer to section 6.3.1.19, "TC_CONTROL_SYNC_LEVEL."

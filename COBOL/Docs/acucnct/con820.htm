8.8.2.3 TC_CONTROL_SYNC_LEVEL runtime configuration variable

Normal runtime behavior ensures that VALUE data items in a Screen section are updated whenever a BEFORE, AFTER, or EXCEPTION procedure executes. This can result in decreased performance under the thin client. You can determine which VALUE data items are updated by the setting of the TC_CONTROL_SYNC_LEVEL configuration variable. Limiting updates can mean improved performance. For more information about this configuration variable, refer to section 6.3.1.19, "TC_CONTROL_SYNC_LEVEL."

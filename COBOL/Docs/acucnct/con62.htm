6.1 Preparing the Server

In thin client configurations, the server is also known as the application host. This is where the COBOL application resides and where the application processing is performed. Unlike the clients, which are referred to as "thin," the server is considered the "thick" portion of thin client configurations.

There are three basic steps to preparing the server to act as a thin client host. You must:

  1. Install server software components.
  2. Configure the server.
  3. Start AcuConnect®.

If you want thin client users to run your application over the Internet, there are additional steps you can take. These are covered in section 6.6, "Launching Thin Client Applications From a Web Page."

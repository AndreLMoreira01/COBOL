To add access records using the ODCP

  1. To add an access record using the graphical interface, open the AcuODBC Server Control Panel and open the server access file to which you want to add a record.
  2. Click New. The Access User screen appears.
  3. Enter the Client Machine Name. You have two options:
    • Enter "*" to indicate that the record will match all clients for which there are no other records. (This will be the default setting for all clients that do not have a specific record.)
    • Enter the official machine name for a client machine. You cannot use alias names in this field.
  4. Enter the Client User Name. If the field is blank, any user name will match. For Windows clients that do not define a USERNAME or USER environment variable, this field should be left blank or set to "USER".
  5. Enter the Local User Name. This is the name that AcuODBC Server will use when executing access requests for requesters that match the first two fields of this record. Note that if the Local User Name is not a valid name on the server, the connection will be refused (AcuODBC Server error 9D,103). Leave the Local User Name field blank if the local name is the same as the client user name.
  6. Enter a password for the user. Use of a password is optional. Passwords may be up to sixty-four characters long. The set of allowable characters includes: upper and lower case letters, numbers, the space character, and most special characters (all ASCII characters numbered 32-126). Delete, escape, and other non-printable characters are not allowed.
  7. Confirm the password (if any).
  8. Enter a umask value. This is an octal value between 000 and 777. The umask is not used by AcuODBC Server, but because other Acucorp products use umask, the information is provided here.
  9. Click OK to accept the record.

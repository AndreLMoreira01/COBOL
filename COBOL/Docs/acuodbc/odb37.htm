Data Directory

This field specifies the directories AcuODBC searches for data files. It is equivalent to the FILE_PREFIX configuration variable.

Until you specify a data directory, AcuODBC uses the same path specified in the XFD Directory field. This means that if your XFD files and data files are in the same directory, you can leave the Data Directory box blank. If your data files reside in a different location than the ".xfd" files, however, you must enter the correct path in the Data Directory box. You can specify multiple locations for your data files.

To complete the Data Directory field:

  1. Enter the name of a directory you want to include in the search path. You can type a path or use the Browse button to locate the directory. Note that if you select the directory using the Browse button, you must click OK. Double-clicking the name of the directory does not insert the name in the entry box.

    2. The path appears in the entry box of the Data Directory field.

  2. Click Add. The path to the directory now appears in the lower box in the Data Directory field.
  3. Repeat these steps for each directory you want to add to the search path. The paths appear in the box in the order in which you enter them.

Here's a quick way to add entries to the search path. Double-click on a path already listed that is similar to the path you want to add. When the path appears in the Data Directory edit field, make any necessary changes to components of the path, and then click Add. The new path appears at the bottom of the list of directories in the search path.

For best performance, list the directories in the order in which you want AcuODBC to search them, using the Move Up and Move Down buttons to rearrange the search order. To delete a directory from the search path, select that directory and click Delete.

9.4 Configuring Remote Data Sources in a UNIX Graphical Environment

The srvconfig.tcl utility enables you to configure your remote DSN in a graphical UNIX environment. If you prefer to use a command line interface, please see section 9.5 for instructions on working with the srvconfig.sh utility.

Both UNIX utilities have been designed to closely mirror the AcuODBC Configuration property sheet, described in Chapter 3. Please refer to that information as you configure your remote data source using either srvconfig utility. Also refer to section 9.3, "Configuring the Remote DSN on a Windows Server," for additional information.

Note: srvconfig.tcl requires Tcl/Tk version 8.0 or later, which is not supplied by Acucorp. See www.scriptics.com for information on obtaining Tcl/Tk. Tcl/Tk is open source software, so the only cost is the time to download.

Also, remember that UNIX is case sensitive; enter paths and filenames as they appear in the file system.

Because srvconfig.tcl is so similar to the AcuODBC Configuration property sheet, this section provides the command to start the utility and highlights any differences between the utility and the property sheet that you should know. Refer to Chapter 3 and section 9.3, "Configuring the Remote DSN on a Windows Server," as you create remote DSNs using srvconfig.tcl.

You can configure srvconfig.tcl to display a Help button for context-sensitive help in HTML. Open the srvconfig.tcl script (most likely found in the "acugt/bin" directory). Near the top of the script, you must set two configuration variables: BROWSERNAME and HTMLDIR.

  • Set BROWSERNAME to the name of the browser you are using. If the browser is in your PATH, you need simply to provide the name of the browser. If the browser is not in your PATH, you must provide the full path to the browser.
  • Set HTMLDIR to the name of the directory that contains the HTML help files. This can be either "Docs7x/Acuodbc/help" (which is where the files reside on the CD) or another directory where the files are installed.

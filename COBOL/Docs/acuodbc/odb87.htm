To create or open the server access file using the ODCP

  1. Make sure that you are logged into your Windows server with Administrator privileges and launch the AcuODBC Server Control Panel using any of the following methods:
    • Go to Start/Programs/Acucorp 7xx/AcuODBC/AcuODBC Server and select AcuODBC Server Control Panel.
    • Type acuodbcs with no options at the Windows command prompt.
    • Navigate to the \Acucorp\Acucbl7xx\AcuGT\bin folder and double-click on the acuodbcs icon.

    A graphical screen appears with the Access tab selected. The Access tab provides graphical access to the access file manager, allowing you to add, modify, and delete user files.

  2. Click Open to select a server access file to manage.
  3. Browse to the directory where the existing access file resides, or to the directory where the new access file will reside.

    4. The default directory for the server access file is "c:\etc".

  4. Select the name of the server access file that you want to manage or enter a new file name and click Open.
    • If you choose an existing file, the contents of that file appear in the Access tab. You can now use the New, Modify, and Delete buttons to add, change, or remove user access records from the access file.
    • If you are creating a new server access file, a dialog box appears, confirming the name and location of the new access file. If the information is correct, click Yes.

      • You can now click New to add new user access records to your server access file. Once you have added records, you can click Modify to change user information or Delete to remove records.

Refer to section 8.3.2, "Managing AcuAccess Records," for more information about the various considerations involved in setting up user access records for AcuODBC Server.

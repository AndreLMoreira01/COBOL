Best Key Match WHERE - ORDER BY

This option is used when an SQL Select Statement contains both WHERE and ORDER BY phrases. It determines which phrase takes precedence when the statement is processed. This check box is selected by default, causing the WHERE phrase to be processed first if it contains fields that make up a Vision index key. The returned record set is sorted before it is returned to the calling directory. If you want the ORDER BY key field to take precedence when searching for Vision records, uncheck the box to turn the option off. Then the files are read in key order, avoiding the additional sort step.

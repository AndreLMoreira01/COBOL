Components for Remote SQL Processing

The elements of AcuODBC for remote SQL processing are:

  • the Microsoft ODBC manager
  • the ODBC driver for ACUCOBOL-GT data (ACUODBC.DLL)
  • the AcuODBC Server executable on the remote processing server ("acuodbcs.exe" on Windows or "acuodbcs" on UNIX)
  • the data dictionary generated by your ACUCOBOL-GT compiler
  • WSOCK32.DLL (This file, which is installed along with the TCP/IP software, must be compliant with the Windows sockets API.)
  • ACUCOBOL-GT data files (Vision or relative)
  • AcuServer (licensed from Acucorp), if necessary (AcuServer is required only if the data files reside on a machine other than the client or the server where processing is taking place. Acucorp strongly recommends, however, that the files reside on the same server as acuodbcs.)

See "Components for Local Processing" for additional information.

Read Only

This option is available only for read/write versions of AcuODBC. This entry establishes the default read/write permission for all the files belonging to this DSN. Select the check box (the default) if the files are read-only. Clear the check box if the users can write to the data file (that is, if the files are read/write). Field (column) level read/write protection can be further defined by adding the HIDDEN and READ-ONLY directives to the file descriptor (FD) for your COBOL program. Refer to section 6.3.6, "HIDDEN Directive," and section 6.3.9, "READ-ONLY Directive," for instructions.

Note: When executing a SELECT statement, AcuODBC opens Vision files as INPUT regardless of the how this option is set. When the user attempts to update a row (from Microsoft Access for example), if the AcuODBC Read Only configuration setting is off and the appropriate license is present, AcuODBC attempts to open the Vision file for update. If the COBOL program has the record locked during an AcuODBC SELECT and that record has met the SELECT conditions, a locked record condition will occur. When the COBOL program updates a record previously retrieved during an AcuODBC SELECT, the update takes effect. When AcuODBC attempts to update the record, an error will occur indicating that the record has been changed by another process after the original AcuODBC SELECT.

What you indicate for read/write permission here applies to all files in the data source. If you want to assign different permissions to any individual files, you can do so using the File Alias tab. See section 3.7, "File Aliases," for more information.

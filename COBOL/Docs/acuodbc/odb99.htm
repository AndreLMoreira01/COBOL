Starting srvconfig.tcl

  1. Change to the directory where you store the DSN information.

    2. Note: Unlike either the Windows server configuration utility or srvconfig.sh, srvconfig.tcl assumes that all activity is occurring in the current directory. Therefore, if you are creating a new DSN, you must change to the directory where you want to store that DSN's configuration information.

  2. To start the UNIX graphical configuration utility, enter the following on the command line:
    3. wish srvconfig.tcl

    The AcuODBC Server Configuration dialog box appears.

  3. Click Add to create a new DSN. The AcuODBC Server Configuration property sheet appears.

Note the following significant differences from the Windows version of the property sheet.

  • The "tabs" in the Windows property sheet are represented by buttons in srvconfig.tcl.
  • Date verification is not in effect in this release. Be sure to double-check the entry in the Julian Base field on the Advanced tab.
  • On the File Alias and Multicompany sheets, the list of entries grows as you enter data and click Add. This is slightly different than the Windows version, where you see a grid when you open the sheet. Select or clear the box at the start of a line to include an entry in an alias or a multicompany DSN. An entry is included when the box is red. See section 3.7, "File Aliases," and section 3.8, "Multicompany Support," for a complete explanation of the fields on these tabs.
  • The NT Setting field does not appear on the AcuServer tab. NT Settings only apply in Windows environments.

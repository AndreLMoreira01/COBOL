AcuServer Port

Enter the port number assigned to the AcuServer host if it is different from the default port number, "6523". This field is useful for redirecting additional instances of AcuServer or for working around a firewall.

The Ping button can help you in troubleshooting, if necessary. If AcuODBC is not connecting with the remote server containing the data files, do the following:

  1. Open the AcuODBC Configuration property sheet.
  2. Select the AcuServer tab.
  3. Click Ping.

AcuODBC attempts to ping the first server it finds in the XFD Directory and Data Directory list of directories. If you can reach the server where the data files reside, you receive the following message: 

Server is alive and responding.

Note, however, that this message indicates that the server is available. AcuODBC does not check the path to see if you are pointing to a valid directory on the remote server.

If the server is not responding, you can receive one of the following errors: 

Please enter a valid port number.

The port number is not a numeric string. If you have changed the port number from the default of "6523", enter a numeric value for the port number. 

No server was found in the XFD Directory or Data Directory 
listing.  Please enter the name of a server, using remote 
syntax.

You can receive this message for one of two reasons:

  • A server is not listed in these fields on the General tab of the AcuODBC Configuration property sheet; only local directories. Check the XFD Directory or Data Directory fields on the General tab to ensure that at least one of the directories listed is remote.
  • A server is listed in these fields on the General tab of the AcuODBC Configuration property sheet, but it is not indicated with remote syntax. Re-enter the name of the server, using remote syntax. See "XFD Directory" in section 3.2 for additional information.
The server refused the connection.  Make sure the port number 
is valid, check the status of your network, or contact your 
network administrator.

Most likely, the server is not up and running. Check the status of your network or contact your network administrator. 

Socket error.  Check the status of your network or contact your 
network administrator.

This message indicates than an error, most likely an interruption, occurred in transmission. Try again to establish a connection, check the status of your network, or contact your network administrator. 

You must run the same version of AcuODBC and AcuServer.  
Contact Acucorp Technical Support.

If you are running a Version 7.2 or higher release of AcuODBC, you must also be running a Version 7.2 or higher release of AcuServer. Check with your network administrator or contact Acucorp Technical Support.

Another class of errors occurs even more rarely. Should you receive an error message that mentions the "acme.dll" or "aclnt.dll" files, note the text of the error message and contact Acucorp Technical Support. These files are needed for further troubleshooting.

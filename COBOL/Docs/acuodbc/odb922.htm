To configure the master DSN using srvconfig.sh

The initial steps involved in creating a master DSN are the same as the steps involved in creating any DSN, as shown in the following sample session. 

$ srvconfig.sh

files [.]?  This should match the value of DSN-DIRECTORY in 
your configuration file.

testing.dsn

Your current Data Directory values are: 
 
Select a command 
Add, [Quit]: q

When specifying General information for a master DSN, specify only the XFD Directory; you will specify data directories when you provide Multicompany information. 

Data File Extension [ ]:

Enter the extension for your data files or accept the default, then add Multicompany information as appropriate. The following steps demonstrate how to list individual DSNs, the location of their data files, and their write permissions.

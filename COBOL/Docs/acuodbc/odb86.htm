To create or open the server access file from the command line

  1. Use the "acuodbcs -access" command to launch the access file manager from the command line. You must be logged in as root or superuser if you are using the UNIX platform. If you are using the Windows server, you must be logged in as a member of the Administrators group.

    2. The access file manager then prompts you for the path and name of the server access file.

  2. Enter the path and file name for the access file that you want to create or edit. To accept the default value, simply press Enter.

    3. In the following example "/etc/AcuAccess" is the default value. 

    Enter the name of the Server Access File 
Filename [/etc/AcuAccess]:

    Note: When the access file manager displays a value inside a pair of square brackets, that value is the default value for the field.

  3. If you specify the name of a file that does not exist, the manager will ask you if you want to create the file.
    4. 'new_file' does not exist. Do you want to create it [N]?
  4. After opening the access file, the manager displays a menu of five options:
    5. Server Access File Options 
1 - Add a security record 
2 - Remove a security record 
3 - Modify a security record 
4 - Display one/all security records 
5 - Exit 
Enter choice [4]:

    You can now view or edit the server access file as described in section 8.3.2, "Managing AcuAccess Records."

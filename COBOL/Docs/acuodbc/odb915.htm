Enter the new XFD Directory: /usr/test/xfds

The XFD Directory entry is case-sensitive. Enter the components of the path as they appear in the UNIX file system. Note that the names of XFD files that reside on the UNIX server, must be completely lowercase letters, such as "accounts.xfd."

srvconfig.sh supports multiple XFD directories, as well as multiple data directories. Select Add as many times as necessary for the number of directories to be listed in this field. 

Your current XFD Directory values are: 
1) /usr/test/xfds 
 
Select a command 
Add, Delete, [Quit]: a

Note that once you have added one directory, Delete appears as an option. 

Enter the new XFD Directory: /etc/xfds 
Your current XFD Directory values are: 
1) /usr/test/xfds 
2) /etc/xfds 
 
Select a command 
Add, Delete, Modify, MoveUp, MoveDown, [Quit]: moveup

For performance reasons, your XFD directories should be listed in the order in which you want AcuODBC to search them. Once you add more than one directory, you have the option of changing the search order. 

Enter the directory to move up (between 1 and 2) [1]: 2

If you do not want to change the order of the directories, enter 1 or accept the default. 

Your current XFD Directory values are: 
1) /etc/xfds 
2) /usr/test/xfds 
 
Select a command 
Add, Delete, Modify, MoveUp, MoveDown, [Quit]: q 
 
Your current Data Directory values are:

(If you are modifying an existing DSN, data directories are listed here.) 

Select a command: 
Add, [Quit] a

If this is a new DSN, select Add, and enter the path and name of the directory that contains the source data files. As with XFD directories, select Add as many times as necessary to add all data directories. 

Enter the new directory: /usr/test/data

You must enter values for the data directory. This is in contrast to the Windows versions where the data directory defaults to the same location as the XFD directory. Note that this is a case-sensitive field. 

Your current Data Directory values are: 
1)  /usr/test/data 
 
Select a command 
Add, Delete, [Quit]: a

"Delete" now appears as a choice. You can also add another directory. 

Enter the new directory: /usr/test/book 
Your current Data Directory values are: 
1)  /usr/test/data 
2)  /usr/test/book 
 
Select a command 
Add, Delete, Modify, MoveUp, MoveDown, [Quit]: movedown

You now have the option of changing the search order of these directories. 

Enter the directory number to move up (between 1 and 2) [2]: 1

If you do not want to change the order of the directories, accept the default. The default is the last assigned number, because that directory cannot be moved down. 

Your current Data Directory values are: 
1)  /usr/test/book 
2)  /usr/test/data 
 
Select a command 
Add, Delete, Modify, MoveUp, MoveDown, [Quit]: q

Security Method

The setting of this field indicates how the client attempts to access AcuServer. The setting of this field must match the setting of the SECURITY_METHOD AcuServer configuration variable. (This configuration variable replaces NT_SECURITY. See your AcuServer documentation for information about this variable.) This field is valid in Windows NT, Windows 2000, and UNIX servers that use shadow password technologies. It does not apply if AcuServer is running on a UNIX file server that uses pluggable authentication module (PAM) libraries.

If the data source is on a supported server, select one of the following security methods:

If ... Then ...

You want to use Windows NT's security system by connecting to the server via a named pipe. This method does not require the use of a password.

Select Named Pipe. The password field in the AcuAccess file is ignored. The AcuODBC client connects to the file server using a named pipe. This gives the client all the features of Windows NT file permissions. The client machine must have permission to create the named pipe. If the client does not have permission, both the client and server systems hang. See your Windows NT documentation for information about permissions and named pipes. Note that Acucorp does not encourage the use of named pipes.

You want to use Windows or UNIX security to log the user into a domain with a password. The password may be stored in the AcuAccess file or can be entered by the user directly.

Select Logon. When "Logon" is selected, AcuServer attempts to log the user onto the domain using the password in the AcuAccess file. If there is no password or the password is not valid, the user is prompted to enter a password. "Logon" is frequently a simpler, more reliable connection method than a named pipe.

See section 8.3, "System Security - The AcuAccess File," for additional information.

Note: The Security Method setting on this tab is for connecting to the remote AcuServer server only. If you are using the AcuODBC Server for processing, you must indicate your Security Method settings on the AcuODBC Server tab. See section 9.2, "Configuring the Remote DSN on a Windows Client," for more information on AcuODBC Server and security.

Use DOUBLE for Numeric Key Data

Selecting this field enables you to use a DOUBLE data type for numeric data.

This is useful if your ODBC-enabled application is Microsoft Access and the Regional Settings in Windows 9x use a comma (",") rather than a period (".") for the decimal point.

Under these conditions, when numeric key data is returned as a string (for example, "123456.789"), Access incorrectly compares originally requested keys with subsequent data and determines that the record is deleted. Versions of AcuODBC earlier than Version 5.1 treated this data as DOUBLE. This setting does not have the precision necessary to handle all the digits that COBOL data can have, but does allow Access to handle numeric key data that falls into the range allowed by DOUBLE (approximately 15 digits of precision) correctly.

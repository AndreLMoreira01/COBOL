SECURITY_METHOD

The SECURITY_METHOD variable specifies the security method used when the client connects to a remote Windows or UNIX server.

If you are using a Windows server, you specify a security method in the AcuODBC Server Configuration property sheet when you create your remote DSN. You do not need to manually edit your AcuODBC Server configuration file. For more information about specifying a security method on Windows servers, Chapter 9, section 9.3, "Configuring the Remote DSN on a Windows Server."

If you are running AcuODBC Server on a UNIX server that supports shadow passwords, you have the option of setting SECURITY_METHOD to LOGON in your "odbcscfg" file. This allows your users to enter their usual password when connecting to the server, instead of having a different password in the AcuAccess file, or having to keep the AcuAccess file coordinated with the system password.

The security method you have specified on the server must match the security method specified on the client. If the values do not match, the security method reverts to NONE, and the AcuAccess password is used. For more information about specifying a security method setting on your AcuODBC client machines, see Chapter 3, section 3.9, "Accessing Remote Files."

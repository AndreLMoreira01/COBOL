To install acuodbcs as a service using the ODCP

  1. From the Start/Programs/Acucorp 7xx/AcuODBC/AcuODBC Server menu, select AcuODBC Server Control Panel, then select the Services tab.
  2. Click New to open the Service Properties dialog box.
  3. Enter the port number, the configuration file name, the error file name, the desired level of tracing, and any relevant dependencies.

    4. To specify a configuration file or error file other than the default, use the Browse button to the right of the entry field to select the location and name of the file. If you choose to enter the path and file name manually, note that AcuODBC Server does not verify the validity of the directory or file name that you specify.

    Option Explanation

    Port

    Assign a port number to this instance of the AcuODBC Server program. The port number must be an integer, such as "6524".

    Client applications can be assigned to a particular instance of the acuodbcs service via the Port Number field on the AcuODBC Server tab of the AcuODBC Configuration property sheet.

    acuodbcs can work with privileged port numbers (from 0 to 1023), and with non-privileged port numbers (1024 and higher). Privileged port numbers are useful for external, secure applications.

    Configuration file

    Specify the name and path of the server configuration file. When this option is left blank, acuodbcs looks for the configuration file in its default location: "\etc\odbcscfg".

    Error file

    Specify the name and location of an error log file for this instance of AcuODBC Server.

    Trace level

    0 indicates no tracing; 7 is the highest level of tracing.

    Dependencies

    Specifies any services that must be running before acuodbcs can start. Click Add in the Dependencies box to open a pop-up window that lists services that may be added as dependencies.

  4. Click OK to install the service.

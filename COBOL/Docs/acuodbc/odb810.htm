To add access records from the command line

  1. To add an access record from the command line, start the access file manager with the "acuodbcs -access" command and select option 1 from the main menu.

    2. The manager presents a series of five prompts, one for each field in the record. The first field is the Client Machine Name field. The prompt looks something like this: 

    A value of "*" for client machine name means that this 
record will match all clients for which there are no 
other records.  You cannot use alias names. The name 
must be the official machine name. 
Enter the official machine name [*]:
  2. Enter a client machine name or accept the default value.

    3. The second prompt is for the Client Username. If this field is blank, any user name will match. For Windows clients that do not define a USERNAME or USER environment variable, this field should be left blank or set to "USER". 

    If no client user name is entered it implies any user. 
Enter client user name []:
  3. Enter a client user name or accept the default value.

    4. The third prompt is for the Local Username. This is the name that AcuODBC Server will use when executing access requests for requesters that match the first two fields of this record. Note that if the Local Username is not a valid name on the server, the connection will be refused (AcuODBC Server returns an error 9D,103). 

    A value of 'same as client' for local user name  
means to use the client user name. 
If no local user name is entered DEFAULT-USER is used. 
Enter the local user name [same as client]:
  4. Enter a local user name or accept the default value.

    5. The fourth prompt allows you to specify a password that must be supplied by requesters who match this record. Setting a password is optional.

    The password can be up to sixty-four characters long. The set of allowable characters includes: upper and lower case letters, numbers, the space character and most special characters (all ASCII characters numbered 32-126). Delete, escape and other non-printable characters are not allowed.

    The password characters are not echoed on the screen when entered. You will be asked to enter the password a second time to verify that it was entered correctly. 

    If no password is entered it implies none. 
Enter password []: 
Retype password for verification:
  5. Enter a password or accept the default value (none). If password verification fails, you see the following message:
    6. Mismatch - try again. 
If no password is entered it implies none. 
Enter password []:

    The final prompt allows you to specify a umask. The umask is not used by AcuODBC Server. Because other Acucorp products use umask, however, the information is provided here. 

    The umask defines the file creation mask for all files 
created by this user.  It must be an octal value between 
000 and 777. 
Enter umask [002]:
  6. Enter a umask value or accept the default value. If an invalid umask value is entered, you will see this message:
    7. Invalid value for umask - try again.
  7. After you specify a valid umask, the access file manager adds the record to the server access file. Press Enter to return to the access manager main menu.
    8. Record added. 
Press <Return> to continue...

If you accept all of the defaults when creating the record, the entry will look like: 

Client Machine Name    Client Username    Local Username    Password    Umask 
*                                         <same as client>  <none>      002

This record entry will match any client and allow any user to connect to the server, provided that the user has an account of the same name on the server.

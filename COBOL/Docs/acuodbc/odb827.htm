To install acudobcs as a service from the Windows command line

Use the command "acuodbcs -install" to install acuodbcs as a service from the Windows command line.

Optional arguments to "-install" include:

Option Description

server

Specifies the name of the server machine. If no server is specified, the server is assumed to be the local host.

-depends

Specifies any services that must be running before acuodbcs can start. This option can be repeated as many times as necessary, but it must precede any valid start options. The ServiceDependency phrase is the actual name of the service that AcuODBC Server should depend on.

If you want to add or modify dependencies for a currently installed instance of acuodbcs from the command line, you must remove the service and re-install it with the new dependencies.

You can make AcuODBC Server depend on a service group. To do this, specify the name of the service group, preceded by a plus sign (+). For example, if Alerter is the name of a service group you want to be started before acuodbcs, the command line is: 

acuodbcs -install [server] -depends
   +Alerter [any valid -start options]

most "-start" options            

Any "-start" option other than "-f" can be used for "-install". (Refer to "Starting acuodbcs from the command line" for a list of options.) These options are stored for service startup on this particular port.

Installing a service on a particular port resets all startup options for the service on that port.

If there are no options stored for a service, starting the server on a particular port automatically installs a service on that port and stores the options used. In other words, you can install and run a service with one set of arguments and then occasionally run the service with different arguments by using "acuodbcs -start".

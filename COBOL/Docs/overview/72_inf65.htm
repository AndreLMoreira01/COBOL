Improved Reporting

To simplify system management, AcuServer Version 7.2 includes the following new reporting features:

  • Error reporting on startup. AcuServer now returns a status to the operating system when it starts or fails to start. This eases administration and troubleshooting. Refer to section 8.2.3 of the AcuServer User's Guide for a description of the return codes.
  • New UNIX syslog capability. If your OS supports it, the UNIX version of AcuServer now writes to the syslog facility when informational or error messages need to be reported. If your OS does not support syslog, AcuServer writes these messages to /dev/console instead.
  • New event log capabilities. The Windows version of AcuServer now has the ability to write to the event log of other machines in the network. On Windows, there is a new WINNT_EVENTLOG_DOMAIN configuration variable for this purpose.

    • In addition, messages that used to go only to the local machine event log are now available on all operating systems.

    Refer to sections 4.2.2 and 8.2.2 of the AcuServer User's Guide for more information on syslog and the event log.

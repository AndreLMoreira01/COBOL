12.1 Supporting Concepts

This section describes the basic concepts underlying the standard controls that can be added to a graphical or character-based screen form. Controls are the building blocks of every screen, and each control contains specific attributes and styles that allow you to determine its appearance and behavior. These attributes and styles are known as properties. Both screens and controls have properties, which can be set up or altered through the Property window.

Most control and window properties are listed on the Alphabetic or Categorized tabs in the Property window. The Alphabetic tab lists the properties in alphabetical order, while the Categorized tab groups them by category, such as appearance or behavior. You can customize the defaults that appear for properties in the Property window using the Screen Designer tree of the Tools/Options dialog, as described in section 4.6, "Setting Screen Designer Options."

The third tab of the Property window is the Event tab, which allows you to specify event, exception, and embedded procedures for each control or window that you create.

In most instances, controls are simple to create and manage, but some are more complex. To supplement the information contained in this chapter, please refer to the following portions of the ACUCOBOL-GT documentation set for detailed information about controls, control properties, and the event, exception, and embedded procedures associated with controls:

  1. Book 2, Chapter 3, "Graphical Controls," provides an overview of controls. It also describes the creation and use of bitmap buttons and the general use of paged list boxes.
  2. Book 2, Chapter 4, "Supporting Concepts and Related Issues," contains background material concerning issues related to the creation of controls and screens.
  3. Book 2, Chapter 5, "Control Types Reference," describes individual controls in detail, including styles and properties and how they can be set. It also lists the events that can be associated with each control.
  4. Book 2, Chapter 6, "Events Reference," describes in detail the events that can be generated when you use windows and controls in an event-driven environment.
  5. Book 3, Chapter 5, section 5.9, "Screen Description Entry," contains syntax rules and general rules for the components of a screen description entry.
  6. Book 3, Chapter 6, section 6.4.9, "Common Screen Options," describes options that are common to Screen section entries and the ACCEPT, DISPLAY, and MODIFY verbs. It also includes the phrases used to specify a control's common properties.
  7. Book 3, Chapter 6, section 6.6, "Procedure Division Statements," contains the DISPLAY control-type format of the DISPLAY verb. It also contains syntax and general rules for the use of this DISPLAY verb format.

A general overview of ActiveX and OLE programming issues appears in Chapter 3, section 3.4, "Using ActiveX Controls and COM Objects in Your COBOL Program," in A Guide to Interoperating with ACUCOBOL-GT.

The remainder of this section provides a general description of control properties and events.

3.5.2 Developing Programs That Do Not Use Code Generating Tools

This is the approach that is most like traditional COBOL program development, except that you're working in a sophisticated integrated development environment. This approach foregoes the use of the Screen Designer and other AcuBench tools that generate code. Therefore, these projects do not need or make use of program structure files, nor do they make use of the Structure view of the Workspace window (project files are accessed in the File view). You, and other members of the development team, are responsible for writing all COBOL code. This approach in no way restricts your use of the ACUCOBOL-GT language, or the non-code generating workbench tools, such as the Code Editor's code completion facilities (see section 9.3.8) or the Configuration File Editor, to name only two.

There is nothing unusual about creating a project for the traditional development approach. For step-by-step instructions, see section 6.3.1. There are, however, a couple of details that deserve your attention.

  1. When you create the project, use the "Blank" project template. The "Blank" template does not create a program structure file (which you do not need).
  2. Before adding files to the project, check and adjust the "Create program file" option. This option directs the workbench to create, or not create, a program structure file for each COBOL program added. To check and adjust the "Create program file" option, open the Options dialog by selecting Options from the Tools drop-down menu. Click on the "plus" sign ("+"), to the left of the Environment label, to view the Environment subcategories. Click on the Miscellaneous category to view its options. Locate the "Automatically create a program structure file" label and make sure that the check box is deselected on the corresponding "When adding a source file..." label. Note that the Option settings apply to AcuBench as a whole, rather than to individual projects or workspaces. Therefore it is a good idea to always check the setting of this option before adding a file to a project.

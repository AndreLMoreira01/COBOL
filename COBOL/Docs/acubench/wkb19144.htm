19.9.12.21 Options/Code Generator/Generate Document

The options in this category provide some control over AcuBench code generation. These options include:

Option Description

Generated documents

These options determine which parts of your code are generated when you initiate a Generate action. You can change the default file extension of generated files in the associated entry fields.

Program file (.cbl)

Enable this check box to direct AcuBench to generate a ".cbl" file for the program.

Regenerate tagged area only

Use this option in combination with "Program file" to direct AcuBench to generate only the tagged areas of the program's ".cbl" file.

Working storage (.wrk)

Enable this check box to direct AcuBench to generate a ".wrk" COPY file.

Exclude variables in program file

Use this option in combination with "Working storage" to direct AcuBench to not generate an item in the ".wrk" COPY file that is already declared in the program's Working-Storage section.

Event paragraph (.evt)

Enable this check box to direct AcuBench to generate a ".evt" COPY file for event procedures.

Exclude paragraphs in program file

Use this option in combination with "Event paragraph" to direct AcuBench to not generate a paragraph in the ".evt" COPY file if it already exists in the program file.

Screen section (.scr)

Enable this check box to direct AcuBench to generate a ".scr" COPY file for the screen.

Report section (.rpt)

Enable this check box to direct AcuBench to generate a ".rpt" file for a report.

Procedure division (.prd)

Enable this check box to direct AcuBench to generate a ".prd" COPY file for the program.

Menu paragraph (.mnu)

Enable this check box to direct AcuBench to generate a ".mnu" COPY file for the screen.

Linkage section (.lks)

Enable this check box to direct AcuBench to generate a ".lks" COPY file for the program.

Attributes of generated documents

These options determine the attributes of your generated files.

Set program file as read-only

Enable this check box to cause the permissions on the program file (".cbl") to be set to read-only.

Set COPY book as read-only

Enable this check box to cause the permissions on all generated COPY files to be set to read-only.

Screen Control Focus

Enter the desired value (from 0-4) for the ACCEPT-CONTROL variable in the "ACCEPT-CONTROL value" entry field. For more information about the purpose of ACCEPT-CONTROL, see General Rule 4 in section 4.2.3, "Special-Names Paragraph," in ACUCOBOL-GT Reference Manual.

Generated document format

These options determine whether your files are maintained in one program file or separate files.

Merge to one program file

Enable this radio button to cause AcuBench to maintain all your generated code, including COPY files, in one program file.

Split to multiple copy files

Enable this radio button to cause AcuBench to maintain your COPY files in separate files.

Attributes of Data Set code generation

Set the "Link file close to open" check box if you want AcuBench to automatically generate a Close I/O paragraph for a data set when the Open option is already set. This code generation option is especially helpful if you are upgrading your Version 5.x AcuBench project to Version 6.x or later.

16.1 Introduction

AcuBench includes a set of specialized tools for working with data files and user-defined items. These tools are designed to simplify the creation and maintenance of data-related program elements. Graphical user interfaces, many of which support drag-and-drop editing and syntax checking, make it easy to create, edit, and review data descriptions. At your direction, the information added to these interfaces is used to generate the related code. For an overview of code generation concepts, see section 3.3.

The data-related development tools are collectively referred to as the data design tools. This chapter contains information about the following data design tools:

File Designer

Used to specify the structure of data files used in a project (and to generate FD, SL, SD, and XFD files)

Data Set Designer

Used to specify which data files are be used by each program (and to generate OPEN, CLOSE, and other program-specific data handling code)

Working Storage Editor       

Used to define Working-Storage items

Linkage Editor

Used to define Linkage Section items

In addition, you can launch the ACUCOBOL-GT indexed file record editor, alfred, and the Vision file utility, vutil, from the Tools menu. You can use alfred to view, modify, add, or delete records in an existing Vision indexed file, or use vutil to retrieve information about, rebuild, load, or unload indexed, relative, and sequential files. For a complete description of alfred, see section 3.5 of the ACUCOBOL-GT User's Guide. For a complete description of vutil, see section 3.3 of the ACUCOBOL-GT User's Guide.

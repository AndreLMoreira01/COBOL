4.5.1 Setting General Code Editor Options

General options for the Code Editor let you determine Line Number pane width, Code Editor line length, ASCII code settings, and behavior of the editor's vertical block select function.

You can set the following General Code Editor options as needed:

  1. Select the General category of the Code Editor options.
  2. "Line number pane width" option: To modify the width of the Line Number pane from the default (4), enter a new value (from 1 to 6) in the "Line number pane width" text box.
  3. "Line length" option: To modify the length of a line in the Code Editor, enter a value up to 2048 in the "Line length" entry field. The default value is 318.

    4. Note that line length may matter if you are copying code from the Code Editor to another software application or editor. If you have lines of the default length (318 characters), for example, and copy text from the Code Editor into an older text editor, even if no individual line of code has more than 80 characters, each line will still be 318 characters long in the new editor (if the editor doesn't automatically trim trailing spaces).

  4. End of line options: If you plan to save the file in an MS-DOS environment, select Save Record Delimiter as CR/LF. With this action, your file's end-of-line marker is the carriage return/linefeed (CR/LF) ASCII code used in MS-DOS. If you plan to save your file in a UNIX environment, select Save record delimiter as LF. With this action, your file's end-of-line marker is the linefeed (LF) ASCII code used in UNIX.
  5. "Virtual Space" option: The "Virtual Space" setting affects the behavior of the Code Editor's vertical block select function (invoked when you hold down the Alt key and drag the cursor). If you want the column selected to be precisely the width of the last line of the text block, clear this check box. To allow greater flexibility in determining the width of the column selected, select this box.

14.2.5 Using Group Headers and the Report Occurs Control

In this report, you insert breakpoints into the report in the form of group headers. When a group header is used, breakpoint logic is automatically generated into the report (".rpt") file. In this report, entry fields are positioned in group header sections. You also use a report occurs control, which lets you use data items described with OCCURS clauses in the report.

After you set up your project and work environment as described in section 14.2, use the following instructions to finish creating this report.

Create and refine the data set that corresponds to your data layout:

  1. Create a new blank program named Report1c from the File/New/Program interface.
  2. Set up the SalesdataDS data set for this program, as you have with previous sample reports.

Design your report in the Report Composer interface:

  1. Open a new blank graphical report called Report1c from the File/New/Report interface, or right-click on the workspace report node and select New Report.
  2. Use the Drag and Drop interface to drag the same four variables named in previous samples into the Report Composer window. Rename the report labels as before.
  3. Add a report date time control to the Page Footer section, as described for the previous report.
  4. Create group headers using the Section Controller.
    • Right-click in the Report Composer window and select Section Controller. The Section Controller contains three areas in which you define header and footer sections, detail sections, and group sections. In the Section Controller's Group Section, click the New (sunburst icon) push button. Click inside the new line's Grouped By cell to display an ellipsis button that, when clicked, opens the Select Variable interface. In the Category drop-down box, select Salesdata. Double-click on Sales-State.
    • Use the procedures described in the previous paragraph to add a second group header, but double-click on Sales-Branch-No in the Select Variable interface instead of Sales-State.
    • Ensure that the Lines property for each group header is set to "0.5".
  5. Move the labels and entry fields into the group header sections.
    • Position the Sales-State report label/entry field combination in the report section called "Group Header: Sales-State OF Salesdata (Field)", which is the primary group header.
    • Move the Sales-Branch-No report label/entry field combination into the report section called "Group Header: Sales-Branch-No OF Salesdata (Field)".
    • In the Property window, change the Border Style property setting for all report entry fields to "No Box".
  6. Create a report occurs control.
    • In the Drag and Drop interface, select "Rpt.Occurs" from the right-hand drop-down box. This determines the control created by the Drag and Drop function. Note that only variables with OCCURS clauses appear in the Drag and Drop window.
    • To create a three-column report occurs control, select the Note-Date, Note-Initials, and Note-Text variables in the NOTES group variable and drag them to detail section in the Report Composer window.
    • Select the report occurs control in the detail section. In its Property window, click the Columns' Setting property's Value cell. Open the Columns' Setting dialog from the ellipsis push button. The Dominant Occurs Data Item appears at the top of this dialog. You can change the default column headings by clicking in the Head column in this interface and typing new ones.
    • Change the Show Grid Line Property window setting for the report occurs control to "FALSE".
  7. Generate the program. A report (".rpt") file and a Report Composer Section in the source (".cbl") file are created.

Make one-time modifications to the generated code:

Use the instructions for code modifications described for sample Report 1a. Note that the PERFORM . . . DO-PRINT-RTN statement is automatically generated by AcuBench.

Compile and execute the program.

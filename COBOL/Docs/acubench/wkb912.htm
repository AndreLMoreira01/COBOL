9.3.1 Using State-dependent Operations

Because the workbench integrates most primary development activities into a single program and user interface, many common operations have different results depending on the current state of the workbench. Therefore, it is important to be aware of the workbench's current state. The current state is determined by the document, facility, or activity that has focus.

State-dependent operations include Open, Save, Close, Undo, Redo, Cut, Copy, and Paste. When using these operations, to be sure that you get the desired result, you must be aware of the workbench's current state before initiating the action. For example, if you are using the Code Editor and you initiate a Save action, the file that you are currently editing is saved (this can be verified if you select Save from the File menu and check the name that appears to the right of the Save option). However, if you switch focus to a screen form to check a variable name, and select Save while the screen form has focus, AcuBench saves the program structure file (the file in which screen descriptions are saved) and not your COBOL source file. Use the Save All option to save all open files.

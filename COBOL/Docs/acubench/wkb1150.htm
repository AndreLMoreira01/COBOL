11.10 Copying Graphical and Character-based Screens

The Screen Designer allows you to copy a graphical screen into a character-based environment or a character-based screen into a graphical screen form. You can also copy individual controls from one screen type to the other. Because generated Screen section code for a graphical screen is labeled GRAPHICAL and character screen code is labeled CHARACTER, your application can contain a graphical screen that has the same name as a character screen. The code that is executed depends on the environment. A graphical environment ignores CHARACTER code, and the character-based environment ignores GRAPHICAL code.

You should keep in mind that when you copy one type of screen into the other type, the resulting screen does not look like the original, even though the code is essentially the same for both screens. Graphical screen elements are measured by pixel, and character screen elements are measured by cell, so you probably need to adjust the new screen after the copy procedure is completed.

Copy one screen type into another screen as follows:

  1. Open the screen you want to copy.
  2. Open a new, blank screen of the other type (graphical or character) using the File/New command.
  3. Make the screen to be copied the active window. Use the Edit/Select All command to select all the components of the screen, then use the Edit/Copy command.
  4. Make the blank screen the active window. Use the Edit/Paste command to paste the copied screen into this screen.
  5. Adjust the elements of the new screen until it is visually acceptable.

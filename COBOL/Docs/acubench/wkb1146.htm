11.8.2 Creating a Pop-up Menu

Pop-up menus appear over the application in a vertical orientation while the user is selecting an item, and then they disappear from the screen. You can programmatically invoke a pop-up menu. In addition, you can arrange for a right mouse-click to automatically invoke a pop-up menu tied to a window or a control. Each window and control can have its own pop-up menu. In addition, you can share a single menu between several windows and controls.

Use the following steps to create a pop-up menu:

  1. Put a menu on a screen using the Menu Designer, and make the Property window visible if it isn't already. Click once on the Menu Designer control icon to highlight it without displaying the Menu Designer window. In the Property window, select either the Static or Pop-up styles for the menu.

    2. Some controls have Pop-up Menu entries in the Property window, with a combo box in the value field. If you open the combo box, you see all the handles of the menus you created and identified as pop-up menus.

  2. The Property window for the screen form contains an additional entry for a Main Menu, which also has a combo box in the value field. If you open the combo box, you see the handles of the menus you created and identified as static or pop-up.

    3. When you select a MAIN MENU handle, the generated code causes this menu to display at the top of your window. The code is put into the ".cbl" program generated. For example, if a screen is called form1 and the static menu handle is FORM1-MN-1-MENU, the code in "form1.cbl" is: 

        PERFORM BUILD-Form1-MN-1-MENU.

    This code generates "form1.mnu", which contains: 

        CALL "w$menu" USING WMENU-SHOW, MENU-HANDLE.

    Pop-up menus do not display via calls to W$MENU like this. They show up automatically when you right-click on the control. The code that is generated for the pop-up menu is just a single line of code in the "form1.prd" file, as follows: 

        MOVE menu-handle TO Form1-MN-2-Handle

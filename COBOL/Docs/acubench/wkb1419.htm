14.2.10 Using the Report Grid Control

This sample report illustrates the use of the report grid control. In this case, our four entry fields in the detail section of the previous report are replaced by a grid control with four columns corresponding to the entry fields.

Set up your project and work environment as described in section 14.2, and use the following instructions to finish creating this report.

Create and refine the data set that corresponds to your data layout:

  1. Create a new blank program called Report1h from the File/New/Program interface.
  2. Set up the SalesdataDS data set for this program, as you have with previous sample reports.

Design your report in the Report Composer interface:

  1. Use the same report template file that you used in the previous sample. Right-click on the Report1h program node and select the Add Report command. Navigate to "Report1f.wtf" in the Add Report to Program dialog (in the AcuGT/sample/reports directory) and add it to the Report1h program.
  2. Add the event code from the original report. In the workspace Structure view, right-click the original report's Event Paragraph to open the Event Editor. Copy the before and after print procedures associated with the report and paste them into the Event Paragraph for the new report you created.
  3. In the report Property window, change the report (Name) property to "Report1h". Change the Output File property to "Report1h.html".
  4. Because a template based on Report 1f has been used, you will notice that the sections and report controls in the report have names based on Report 1f. For example, the page header section name in sample Report 1h is "Report1f-PH-1". Change these names to reflect Report 1h in the Property window for each report section and control.
  5. Delete the four report entry fields in the detail section and replace them with a grid control that has four columns. In the Drag and Drop interface, select "Rpt.Grid" from the right-hand drop-down box. Select the following variables and drag them to the detail section in the Report Composer window: Sales-LastName, Sales-FirstName, Heavy-Equipment-Sales, and Supplies-Sales.
  6. Click the value cell of the Columns' Setting property and click the ellipsis push button to open the Columns' Setting dialog. Change the Sales-LastName settings so that width is 1.50 and alignment is Center. Change Sales-FirstName to have a width of 1.50 and a Center alignment. For Heavy-Equipment-Sales, change the Name column to "PR-Heavy-Sales", its PIC to $ZZZ,ZZ9.99, its width to 1.50, and its alignment to Right. For Supplies-Sales, change the Name column to "PR-Supplies-Sales", its PIC to $ZZZ,ZZ9.99, its width to 1.50, and its alignment to Right.
  7. In the report grid's Property window, set the Column Headings property to "FALSE". To ensure that the grid lines are joined, set the grid's Lines property to 0.18 inches and the detail section Lines property to 0.19 inches.
  8. Generate the program. A report (".rpt") file and a Report Composer Section in the source (".cbl") file are created.

Make one-time modifications to the generated code:

  1. Use the instructions for code modifications described for sample Report.

    2. Note that the PERFORM . . . DO-PRINT-RTN statement is automatically generated by AcuBench.

  2. Notice that the Master Print Loop code that is automatically generated also includes program tags related specifically to grid control handling. For this report, you do not need to modify the code in these areas.

Compile and execute the program.

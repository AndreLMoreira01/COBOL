4.4.5 Setting Debug Options

The Debug options allow you to specify whether AcuBench should perform the same save and code generation actions, prior to starting a program in the debugger, that are performed prior to a build/rebuild. The importance of this option is demonstrated in the case where you initiate a debug session and the workbench determines that parts of a program have been modified since the program was last compiled. It is likely that you will want the workbench to first generate code and compile the program before debugging the program. This debug option allows you to specify that the workbench behave in the same way that it does prior to a build/rebuild. Note that if you have selected "Always save all documents" in the Tools/Options/Build interface, you may not want to use this debugger option. These options together cause the debugger to check for source code changes before each step, which could have a significant effect on performance.

You can also determine whether the current line of execution in the debugger is highlighted and what the highlight color is.

To set Debug options, perform the following steps, as needed:

  1. Select the Debug category of the Environment options.
  2. Enable the Follow the options specified in `Build' page check box to direct the workbench to apply the settings in the Build category to the set of actions taken prior to starting a program in the debugger.
  3. Enable the Colorize line of execution check box if you want the current line of execution in your code to be highlighted. Choose a highlight color in the drop-down box to the right of the check box.

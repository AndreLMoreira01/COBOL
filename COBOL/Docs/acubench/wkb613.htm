6.3.1 Creating a Project

When you create a project, AcuBench creates the infrastructure for the project including a set of folders (working directories). Optionally, you can create a project that includes a templated source file, a templated screen, and a program structure file (".psf"). You can also create a project with object files directed to a remote server. For more about projects, see section 3.2.2.

Once a project is created, you can change the name of the project, but you cannot change the name of the project's working directory or the names of any of the sub-directories.

Before you create a project, you need to decide whether to add the new project to the current workspace (if a workspace is open) or to create it in a new workspace. If you want to work on the new project concurrently with another project, open the workspace that contains that project before creating the new project.

Create a new project using the following procedures:

  1. If your project's object files reside on a remote server, select the Build/Use Thin Client command. This command enables you to use thin client related commands and project setting options. Note that the main application window title bar for remote projects includes the phrase "Thin Client Enabled."
  2. Select New from the File menu and click on the Project tab.
  3. Select a project template.

    4. The Standard template creates a project and a program. The program, in addition to the program structure file, gets a standard graphical screen.

    The Blank template creates only an empty project structure.

  4. Enter the name and location of the project in the designated fields.
  5. Select an option to either include the project in the current workspace (this option is disabled if no workspace is open), or to create the project in a new workspace.
  6. If you want to change the name of any of the standard project directories, click the More Info button and make the desired changes in the Project Information dialog.
  7. If you have selected the Use Thin Client command, the remote settings area of the Project Information dialog is enabled for your input. Enter the server name, the port number, and the remote object directory for your project. Click the Validate button to determine whether AcuBench can make the remote connection. If the remote server is up and AcuConnect is running, the validation should succeed. If it does not succeed, you receive the message, "ACUCOBOL-GT cannot connect to servername", where servername is the name of the remote server. If you enter a nonexistent remote directory name, you receive the message, "remotedirectoryname is invalid, or does not exist", where remotedirectoryname is the invalid directory name.
  8. Click OK to create the project.

To add files and resources to your project, click the Add/Remove Files button on the Project toolbar (see section 6.3.4).

Note: You do not open, close, and save individual projects. Every project is a member of a workspace, and it is the workspace that is opened, closed, and saved.

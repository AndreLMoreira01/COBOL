13.7 The File Designer IO Handling Tab

Using the IO Handling tab of the File Designer, you can define and code I/O paragraphs and assign them to a particular data layout file. Because data layout files are defined at the project level, I/O paragraphs that you create here are available to all programs in the project.

The IO Handling tab includes pre-defined spaces for you to add some kinds of I/O code, including declaratives, OPEN and CLOSE code, and transaction handling code. You can also create your own types of procedures, such as the Before Master Print Loop used to efficiently drive reports created in the Report Composer. When you create a new procedure or edit an existing procedure, AcuBench opens the Event Editor, allowing you to add or modify code in this familiar interface.

You can also use the IO Handling tab to choose when and how you want AcuBench to generate I/O handling code. Refer to the section 16.2.5, "The IO Handling Screen," for more information about this interface.

17.9 XML Support: AcuXML

The ACUCOBOL-GT runtime has a file system interface for XML files, called AcuXML, that seamlessly converts XML data to COBOL data and vice versa so that you can trade data with third parties that are not invested in COBOL. XFD files must be created for files to be used with AcuXML. To create XFD files for sequential files, use the ACUCOBOL-GT "-Fx" compile option.

Users can designate a file to be of type ".xml" via the filename_HOST configuration file variable. Alternatively, the DEFAULT_HOST configuration file variable can be set to XML. As a result, all files except those named in user-defined variables are created as type XML, which are line sequential.

For information on working with XML data, please refer to section 8.2, "Working with XML Data," in A Guide to Interoperating with ACUCOBOL-GT.

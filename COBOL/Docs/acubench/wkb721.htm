7.5 Setting File Compile Options

File compile options are the set of compiler options that are unique to an individual file. It is expected that in most cases, individual file compile options will not be necessary. However, in those cases where individual compile options are required, you can use the File Compile Options mechanism to establish independent settings.

To set compiler options for an individual file, use the following steps:

  1. In the File view of the Workspace window, right-click on the file icon of the desired source file and select Program Compile Options from the pop-up menu.
  2. Check the Project tree and "Settings For" values at the left of the dialog to ensure that you are setting options for the desired project and mode. A title field in the dialog contains the name of the project whose settings you are currently changing.
  3. In the upper left of the Compiler tab, ensure that the "Follow project default options" check box is not selected (does not have a check mark). If the check box is marked, you cannot set any options.
  4. Select a category of options from the Catalog drop-down list box.
  5. Set the desired options.
  6. Continue to select categories and set options.
  7. When complete, click OK.

You can also specify any option flag directly in the "Additional options" entry field located to the right of the Catalog drop-down list box. Simply enter the compile flag followed by any required argument. For example: "-Zo C:\program\datadict" specifies the repository directory for data dictionary files.

If you want to force the file to always be compiled with the default project compiler options, enable the Follow project default options check box.

To compare project option settings with those set on an individual file, right-click on the file of interest in the File view of the Workspace window, and select Output Compiler Options from the pop-up menu. The project and file options settings are displayed in the Build tab of the Output window.

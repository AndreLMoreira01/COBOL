7.1 Project Options

AcuBench provides support for specifying and maintaining project level compile, runtime, and environment options.

  • You can define distinct project option settings for each AcuBench project.
  • You can define sets of settings (modes) that enable you to quickly shift to settings that meet special needs, such as settings that support debugging work, or settings that build the program for 64-bit systems.
  • You can save modes that you create, and load the saved modes into other projects.
  • You can establish separate compile options for any COBOL source file.

Project options are divided into three categories: compile, runtime, and environment. You can launch the Project Settings dialog from the Project menu or by clicking the Project Settings button on the Project toolbar. When setting options, you should verify that you have selected the project and mode (set of settings) that you want to change (see section 7.1.1). When you are done, click OK. This causes the new settings to be saved in the workspace file (".prj").

You can also set options on individual files. Once established, the special settings for that file supersede the project settings. The special file options are applied every time the project is built or the individual file is compiled. For more information about file compile options, see section 7.5.

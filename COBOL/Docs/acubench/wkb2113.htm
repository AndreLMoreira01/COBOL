21.1.11 File View: File Pop-up Menus

Right-clicking on a Source file in the File view of the Workspace window creates a pop-up menu containing the following commands:

Menu Item Description

Open program_name

Opens the named program in the Code Editor.

Create CFD

Creates and displays a Control Flow Diagram (CFD) of the program

View Preprocessed

Opens the AcuSQL pre-compiled intermediate file in the Code Editor. This option is enabled only if the source file has been pre-compiled with the option to save the preprocessed file.

Delete program_name

Deletes the named file from the project.

Delete From Disk

Deletes the selected file from the disk.

Compile program_name

Causes the named file to be compiled.

Reparse program_name

Causes the named file to be reparsed.

Execute program_name

Executes the object file associated with the named file, if an executable object exists.

Debug (Runtime)

Starts the ACUCOBOL-GT runtime debugger.

Program Compile Options

Opens the Project Settings dialog, allowing you to set program-specific compile options.

Screen Import

Starts the selected program in the graphical screen import utility.

Character Screen Import

Starts the selected program in the ACUCOBOL-GT Character-to-GUI Wizard.

Output Compile Options

Causes AcuBench to display the file compile options and the project compile options in the Output window.

Find in Objects

Lets you locate all instances of a given string stored in the Property window and the graphical design interfaces, such as the Working Storage Editor or the File Designer.

Properties

Displays the file properties.

Version Control

Displays the list of user-defined version control commands. This item appears on the menu only if version control commands have been defined.

Right-clicking a file in the Screen, Report, Copylib, List, Resource, or FD folder in the File view creates a shorter pop-up menu containing the following commands:

Menu Item Description

Open file_name

Opens the named file in the Code Editor.

Delete file_name

Deletes the named file from the folder.

Delete From Disk

Deletes the selected file from the disk.

Find in Objects

Lets you locate all instances of a given string stored in the Property window and the graphical design interfaces, such as the Working Storage Editor or the File Designer.

Properties

Displays the file properties.

Version Control

Displays the list of user-defined version control commands. This item appears on the menu only if version control commands have been defined.

Right-clicking a file in Object folder creates a slightly different pop-up menu containing the following commands:

Menu Item Description

Delete file_name

Deletes the named file from the folder.

Delete from Disk

Deletes the selected file from the disk.

Execute program_name

Executes the named object file.

Debug (Runtime)

Starts the selected object file in the ACUCOBOL-GT runtime debugger. To start the program in the AcuBench integrated debugger, select Go from the Debug toolbar.

Native Code

Opens the Translate Portable Object Code to Native Code dialog box, which allows you to translate the selected portable object code file into native object code. The translation facility in cblutil is used to perform the translation (see section 17.2).

Find in Objects

Lets you locate all instances of a given string stored in the Property window and the graphical design interfaces, such as the Working Storage Editor or the File Designer.

Properties

Displays the file properties.

Version Control

Displays the list of user-defined version control commands. This item appears on the menu only if version control commands have been defined.

The right-click menu for files in the Remote folder is nearly identical to the menu for local object files (above), but contains one additional option, which appears just after the Debug (Runtime) command:

Menu Item Description

Create Alias program_name

Opens the Create Alias dialog box, which allows you to designate an alias entry name and working directory.

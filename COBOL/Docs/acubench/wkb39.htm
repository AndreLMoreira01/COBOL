3.3.2 AcuBench-generated Code

When you work in any of the AcuBench editors or designers, the changes that you make are stored in the program's structure file. When you generate the program, that stored information is turned into code. By default, a small portion of that code is generated into the COBOL source file (".cbl"), and the rest is added to special, read-only COPY files, as described in the following sections.

Code generated by the Screen Designer

Depending on the content of your screen, screens created with the Screen Designer can automatically generate code for the:

Main program (".cbl")

includes COPY statements for files listed below, code to PERFORM the main screen, and a basic EXCEPTION paragraph

Screen section COPY file (".scr")

contains Screen section definitions

Working-Storage COPY file (".wrk")

includes user-defined screen variables

Menu description COPY file (".mnu")

contains menu definitions

Procedure COPY file (".prd")

includes standard screen handling routines

Event procedure COPY file (".evt")

includes event and embedded procedures coded in the Event Editor

Code generated by the Report Composer

When you create a report using AcuBench's Report Composer, a report COPY file (".rpt") is created and added to the project's Report folder in the File view of the Workspace window. In addition, code is added to the COBOL source file (".cbl") to launch the report-creation process. This is done to provide an easy way for you to add your own code to control how the report is generated. Generated and user-defined report code is discussed in Chapter 14, "Working with Reports."

Code generated by the Event Editor

Code that you write in the Event Editor is generated into the program's event procedure COPY file (".evt"). See section 11.7.3.

Code generated by the File Designer

When you define a data file in the File Designer, AcuBench creates a data layout file (".dlt") to store information about that file, including any customized file-handling code that you have defined. When you generate your program, the data layout file is used to generate COPY files for the SELECT statement (".sl"), copied into the File-Control Section, and for your File Descriptions (".fd"), copied into the File Section. Any special file-handling code that you have defined is also added to the event procedure COPY file (".evt"). If you have defined the data as belonging to a sort file, a sort COPY file (".sd") is generated.

Code generated by the Data Set Designer

Data sets defined in the Data Set Designer generate standard file OPEN, READ, WRITE, and REWRITE routines into the program's Procedure Division COPY file (".prd").

Code generated by the Working Storage Editor

Data items defined in the Working Storage Editor and CRT STATUS values defined in the program's Program Properties dialog, are generated into the program's Working-Storage COPY file (".wrk").

Code generated by the Linkage Editor

Data items defined in the Linkage Editor are generated into the program's Linkage COPY file (".lks").

17.6.3 Testing File Integrity

The vutil test file function is accessed on the Testing File tab. The test file function checks a Vision file for integrity. Integrity is examined on such points as whether a file is corrupt and whether it has a non-zero user count. Testing for file integrity should be one of the first actions taken in assessing data file problems. For a complete description of the vutil test file function, see section 3.3 of the ACUCOBOL-GT User's Guide.You can test a Vision data file's integrity as follows:

  1. Specify a data file name in the "Data file" entry box. You must specify a file name to enable the Start button. You can use the ellipsis button adjacent to the entry field to navigate to a file. If during your use of vutil you have already specified the file, you can easily select the file name again from the "Data file" drop-down list.
  2. In the Parameters section, select the options that you want. The "Option" field displays the command line that reflects the options chosen. You cannot directly edit the contents of this field. For a complete description of the file testing options, see section 3.3 of the ACUCOBOL-GT User's Guide.
  3. To initiate testing, click Start. The results are displayed in the lower half of the Testing File tab.

To interrupt file testing that is in progress, click the Stop button (the Start button becomes a Stop button after the query begins).

Click Close to exit vutil.

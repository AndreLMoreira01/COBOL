17.10.1 xml2fd in the Tools Menu

Users can access xml2fd from the Tools menu. When xml2fd is invoked from the Tools menu, it does not know the context of the project, so it cannot prefill the "-d" entry field area in the xml2fd utility interface.

The "-d" option browse button displays an Explorer-style navigator. With this interface, you can select a directory into which the FD and SL will be generated. Use the "-f" and "-s" options to generate the FD and SL into separate directories. The selection of "-d" causes "-f" and "-s" to be disabled because they are mutually exclusive options. Likewise, the selection of "-f" or "-s" causes "-d" to be disabled.

In the absence of any instructions ("-d", "-f", or "-s") about where to write the output, output is written to the current working directory, typically the root of the active project.

Setting the "-n" option tells the utility the number of records to read before it arrives at field descriptions in the FD. In very large XML files, it may not be desirable to read the entire file to derive the FD. On the other hand, it may be necessary to read more than one record to confirm a "best guess" for intended field descriptions in a record. The value of "-n" must be numeric and greater than "0".

With the "-p" option, users can direct xml2fd to add a standard prefix to the data items in the FD.

Setting the "-o" option and designating a numeric value greater than "0" instructs xml2fd to look for cases within the record where it is appropriate to assign an OCCURS clause, and gives a default number to assign in the OCCURS clause. In complex cases where an FD has more than one OCCURS clause and where the number of OCCURS differs, users must manually change the generated representation of the FD in the graphical designer.

The print usage option ("-h") is not supported.

The Options window displays the command line, as it is built with selections from the dialog.

The Output window prints the output of xml2fd. More information about this utility can be found in section 8.2.2, "XML-to-FD Utility," in A Guide to Interoperating with ACUCOBOL-GT.

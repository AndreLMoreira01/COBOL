7.1.1 Modes

In AcuBench the term mode is used to refer to a set of project option settings. Having two or more modes provides a powerful way to quickly switch from one set of settings to another. You can define as many modes for a project as you want. For example, you can use a separate mode to test remote file access versus local file access for your application. Unique settings for thin client environments may also suit your needs. Creating modes can let you compare the effects of different compiler, runtime, and environment settings on program performance.

Any mode that you create can be saved in a file with a ".pof" extension (for project options file) and loaded by other projects.

When you create a project, the project automatically gets two modes: Debug Mode and Release Mode. Just as the names of these modes imply, modes provide an efficient method of applying the different settings needed to achieve different development objectives.

New modes are established in the Project Settings dialog. The left side of the dialog contains a box showing the current active mode and its settings. Click the icon to the right of the "Settings for" entry field to display an "Add/Remove Project Mode dialog" if you want to add or remove a mode. The Project/Set Active Mode command lets you choose the mode you want to be active. If you decide to make use of multiple modes, be very careful to always confirm that the desired mode is selected before initiating a compile, build, or run.

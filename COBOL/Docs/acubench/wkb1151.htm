11.11 Generating a Screen

When you use the Build/Generate command, the Screen Designer automatically generates a COBOL code source file (".cbl") for a screen. The source code format can be ANSI source format or terminal source format, depending on your selection in the Tools/Options/Environment dialog box. ANSI source format is the default.

The code generation process also gives you a Screen section file (".scr"), an event paragraph file (".evt"), a menu paragraph file (".mnu"), a Procedure Division file (".prd"), and a Working-Storage file (".wrk"), as well as a screen form file (".stf").

The Screen Designer code generator creates an ACUCOBOL-GT ".cbl" source code file, which is a skeletal COBOL program that displays and accepts the screen. The name of this source code file is based on the screen name. For example, if the screen name is "entry", the generated COBOL code is "entry.cbl."

The ".scr" file contains the generated COBOL program's Screen section. Having a separate ".scr" file allows developers to merge Screen sections into existing applications, or to put many screens into one COBOL program.

The ".evt" file contains the COPY file for the code that handles all the events for your screen's controls. "Exception," "before," and "after" procedure code also appear in this file.

When a developer creates the menu paragraph, an ".mnu" file is generated. It contains all of the COBOL code for creating a menu and is referenced with the COPY command. If your screen doesn't have a menu when you generate code, an empty ".mnu" file is generated.

The purpose of the ".prd" file is similar to that of the ".scr" file, except that it contains the generated COBOL program's Procedure Division paragraphs. It describes the behavior of the program and contains sentences that are executed in the normal course of the COBOL program.

The Working-Storage file is similar to the ".scr" and ".prd" files, except that it contains the generated COBOL program's Working-Storage variables. This file facilitates the merging of Working-Storage variables into existing applications. The ".wrk" file contains data items that allow the program to detect what action has been performed by the end user. Typically, data items represented in the Working-Storage variables include such information as the value entered into an entry field.

When you save your screen, it is saved as an ".stf" meta file. When you need to modify a screen, all you need to do is open the ".stf" file in the Screen Designer, make your changes, generate COBOL, and integrate it into your project.

Working with external settings

You may have occasions when you do not want a paragraph or variable name generated for a control component. For example, if a subscript value is entered into a variable that is in parentheses, that variable's inclusion in the ".wrk" file can cause a compilation error.

You can indicate specific paragraph and variable names that you do not want generated in your ".wrk" file through the External Settings commands. These commands are available as a right-click function in the Program level of the workspace Structure view. You can also access this function in the Tools/Options dialog box. In the Code Generator/Generate Documents interface, select the Exclude paragraphs in program file or the Exclude variables in program file check box to exclude paragraph or variable names from your ".wrk" file.

You can indicate external paragraph settings as follows:

  1. Right-click on the Program level of the workspace Structure view, and select External Paragraph. This action displays the External Paragraph dialog box.
  2. Check the paragraph names that you do not want generated in your code.
  3. Click OK.
  4. If you want to exclude all paragraph names from your ".wrk" file, select the Exclude paragraphs in program file check box in Tools/Options/Code Generator/Generate Documents interface.

Indicate external variable settings as follows:

  1. Right-click on the Program level of the workspace Structure view, and select External Variable. This action displays the External Variable dialog box.
  2. Check the variable names that you do not want generated in your code.
  3. Click OK.
  4. If you want to exclude all variable names from your ".wrk" file, select the Exclude variables in program file check box in Tools/Options/Code Generator/Generate Documents interface.

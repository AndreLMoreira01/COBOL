3.5.5 Creating or Converting Screens

When it's time to add graphical screens to your AcuBench project, you can choose between creating new screens using the Screen Designer or importing existing screens using one of ACUCOBOL-GT's screen conversion tools. Whether you choose to create a screen from scratch or to use one of the import tools depends on several factors, including how closely you want the AcuBench screen to resemble the original screen (or how different you envision it being), whether the original screen was created by ACUCOBOL-GT in the first place, and how many screens your application contains, to name a few.

To convert character-based ACUCOBOL-GT screens into graphical ones, you use the Character-to-GUI Wizard. The wizard watches a running character application and, at a specified point, constructs an equivalent graphical screen. This screen is automatically imported into the AcuBench Screen Designer where you can make modifications to it as desired. You can then use AcuBench to produce a Screen section description of the graphical screen and integrate the new Screen section into your original program. The wizard is described in section 11.3 in this manual.

To import a graphical ACUCOBOL-GT screen into AcuBench, you use the Screen Import Utility. This utility creates a text file that defines a graphical screen created by an ACUCOBOL-GT program and imports the screen into the AcuBench Screen Designer. The Screen Import Utility is described in section 11.4 in this manual.

Importing screens is an excellent way to transition into the AcuBench environment. It allows you to lessen your initial development load and provides a rapid implementation path for new users. However, if you choose to import existing screens, be prepared to spend time manipulating the screens to your liking once they're in the Screen Designer. In particular, if you are converting character-based screens using the conversion wizard, the result is not very graphical. The Character-to-GUI Wizard creates a graphical equivalent of the character-based screen and does not automatically add new graphical controls or modify any attributes. Because it mimics the character interface, the resulting graphical interface makes use of only labels and entry fields. To make your screen truly graphical, you should plan to spend time adjusting the screen design after it is in AcuBench.

Creating new screens, on the other hand, gives you the flexibility to begin with a clean slate. You can design the screen from the outset to have the usability characteristics you desire. You are not constrained in any way by the original design of your screens.

Whichever approach you decide to take, AcuBench simplifies your task by offering a wide array of Screen Designer options.

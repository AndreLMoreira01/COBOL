Adding COPY Files With Auto-parsing

  1. Add source files using the procedure outlined in section 6.3.4. Before clicking OK, confirm that the "Automatically parse program for copy files" check box is enabled. Note that the Copy Path button is enabled.
  2. By default, COPYPATH is set to the relative paths of the project's screen, COPY, resource, and layout (FD) folders, as well as the absolute path to the ACUCOBOL-GT sample directory and its ".def" (COPY files) folder ("C:\Acucorp\Acucbl720\AcuGT\sample\def", for example). To redefine COPYPATH to include other directories, click the Copy Path button. The Add Copy Path dialog box is displayed.
    1. Using the mouse, navigate to the desired directory in the Directory list box and then click Add. The new path is displayed in the "Default copy path(s)" list.
    2. Repeat this procedure for every directory that you want to add to COPYPATH.
    3. When you are done, click OK. The Add Copy Path dialog is closed, returning you to the Add/Remove Files dialog.
    4. You can remove a path from COPYPATH by selecting the path from the Default copy path(s) list and clicking Remove.

      5. Caution: Changing the definition of COPYPATH can potentially affect every file in the project. If a directory is removed from the COPYPATH definition and a program in the project has a COPY file located in that directory, that program will fail to compile.

  3. You are now ready to parse the new source files. Click OK. Observe the messages in the Output window confirming that the files have been parsed.
  4. If for some reason parsing fails, a message is displayed and the source file is given a file icon (displayed in the Source tab) that indicates that the parse failed.

Note: You can see the project's parsed COPY files in the File view by expanding the Copylib folder. If for any reason you want to reparse the source file later, you can do so by right-clicking the source file icon (in the File view) and selecting Reparse from the pop-up menu.

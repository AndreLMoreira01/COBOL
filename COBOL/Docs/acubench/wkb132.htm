13.1 Report Composer Functions

This chapter introduces you to the AcuBench components that allow you to develop and format graphical and character reports. You define the appearance and behavior of your report in the Report Composer interface, the Section Controller, the Report Component Toolbox, and the report Property window. The Report Composer window is the primary interface for report design. The Section Controller lets you define the sections in your report, for example, the page header or footer. Like the Screen Component Toolbox, the Report Component Toolbox contains "controls" that correspond to the elements that can appear in a report. You can draw controls in the Report Composer window from the Report Component Toolbox. You can also use the View/Drag-and-Drop command to display the Drag and Drop window. The Property window lets you control the characteristics of the various elements, report sections, and the report itself.

The Tools/Options/Report Writer interface provides options for controlling report control behavior in the Report Composer window and default Property window settings, similar to the options available for the Screen Designer. Refer to section 4.9, "Setting Report Writer Options," for information about these Tools/Options settings.

A File Designer IO Handling tab lets you define any I/O paragraphs that you need and choose the type of automatic I/O paragraph code generation you desire, if any. Other AcuBench functions contribute to the report writing capability. The Generate function for a report results in a report file (".rpt"), which contains most of the characteristics of your report. You must manually insert the master print loop that drives the report. The examples later in this chapter provide sample reports and demonstrate how and where to insert the code needed to complete the report.

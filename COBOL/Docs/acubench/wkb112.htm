11.1 Opening a Screen

The workbench allows you to create new graphical and character-based screens for your project, open graphical and character-based screens that already exist in your project, or open an external screen file in your project. The following sections describe these procedures.

Because the workbench integrates most primary development activities into a single program and user interface, many common operations have different results depending on the current state of the workbench. Therefore, it is important to be aware of the workbench's current state. The current state is determined by the document, facility, or activity that has focus.

State-dependent operations include Open, Save, Close, Undo, Redo, Cut, Copy, and Paste. When using these operations, to be sure that you get the desired result, you must be aware of the workbench's current state before initiating the action. For example, if you are using the Code Editor and you initiate a Save action, the file that you are currently editing is saved (this can be verified if you select Save from the File menu and check the name that appears to the right of the Save option). However, if you switch focus to a screen form to check a variable name, and select Save while the screen form has focus, AcuBench saves the program structure file (the file in which screen descriptions are saved) and not your COBOL source file.

Note: Use the Save All option to save all open files.

You can open a new screen file using the following steps:

  1. Select the File/New command or the New toolbar button and then choose the Screen tab from the New dialog box. You can also right-click on the appropriate program name in the workspace and choose New Screen. This action displays the New Screen dialog box, from which you can select a new screen template.
  2. From the Screen tab, choose a template for your screen. Both graphical and character-based templates are available.
  3. Fill in the Form Name entry field if you want your screen file to have a name other than the default that the Screen Designer supplies. This name appears in the title bar of your screen file.
  4. The "Unique Prefix" entry field is filled in automatically with your screen name. The Screen Designer uses this prefix for its default control and variable naming scheme. Change this prefix if desired. Note that you can also change any screen's unique prefix by right-clicking in the Screen Designer window and selecting Change Prefix. This command displays the Change Prefix dialog box.
  5. Add your new screen to a new program in the existing project or add it to an existing program in your project using the radio buttons at the bottom of the dialog box.
  6. Click OK.

To open an existing screen file, right-click on the screen file that you want to open in the Structure view of the workspace and choose Open.

To open an external screen file, right-click on the Screen icon in the Structure view of the workspace, and select Add Screen. In the Add Screen to Program dialog box, navigate to the desired ".stf", ".smf", or ".out" file and open it. The screen should appear in your project, both in a new Screen Designer window and as an icon in the workspace Structure view.

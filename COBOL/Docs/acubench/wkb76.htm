7.2 Setting Runtime Options

The Runtime tab of the Project Settings dialog allows you to set runtime options for the project. The selected options are applied whenever a program (object file) is run from within AcuBench.

The Runtime tab groups runtime options into four categories. Select a category from the Catalog drop-down list box.

Common options

A variety of commonly used switches

I/O options

Options related to screen, keyboard, and file I/O

Debugging options

Options related to debug operation and output

Thin client options            

Options to name and edit a remote configuration file

To set runtime options, enable the desired options in the Project Settings dialog. If you have selected an option that requires a value, specify the value in the adjacent entry box. When you are finished, click OK.

In the Common Options category, observe that the "Alternate configuration file name" check box is selected by default. This creates an empty configuration file with a default name to which you can add any configuration variables that you require. You can change the path and filename in the entry-field next to the check box to point to an existing configuration file or create a new configuration file in another location. If you do not require a configuration file, clear the check box. To edit the configuration file, click Edit to lauch the Configuration File Editor (CFE). For a description of the CFE, see section 7.3, "Using the Configuration File Editor." For complete information regarding the use of configuration files and variables, refer to Book 1, Chapter 2 of the ACUCOBOL-GT documentation set.

In the Thin Client Options category, when you check the "Remote configuration file name" check box, you enable the "-c" option for remote object execution. You can name a remote configuration file, and the Edit button lets you access the workbench configuration file editing function. Note that the Browse button available in the Common Options category is not available for thin client.

The "Additional options" entry field, located to the right of the Catalog drop-down list, can be used to directly specify any runtime option. Simply enter the runtime flag followed by any required argument. For example: "-e C:\program\errors.log" specifies the file into which runtime error messages are placed.

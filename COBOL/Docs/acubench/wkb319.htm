3.5.3 Developing New Applications

If you are developing a new application and intend to use AcuBench tools that generate code, such as the Screen Designer or File Designer, there are several issues that you should consider before you create your project. Of tantamount importance is that you read and become thoroughly acquainted with section 3.3, "Automated Code Generation." Section 3.3 provides essential information about the AcuBench code generation model and it outlines the associated program organization and structure.

Before you create your new project:

  1. It is easier to incorporate code generating tools at the outset of program development than it is to add them later, although all tools can be adopted later in the development process. It is beneficial to consider and decide at the outset which tools you intend to use.
  2. You should decide whether you are going to include and use tags in the main program (".cbl"). Although tags are designed to make program development simpler and more flexible, you don't have to use them. If you don't use them, you are responsible for including all of the necessary COPY statements that include generated code.
  3. You have significant control over what code is generated by the workbench. Code generation options are set in Tools/Options/Code Generator. These settings apply to all projects and workspaces. Therefore, if you want to quickly change settings to accommodate the needs of a different workspace or project, you need to save the settings to a separate file and load that file when you want to change settings (see section 7.1).
  4. Every program in the project that makes use of a tool that generates code has to have a program structure file. When you create a new project with the File/New/Project command, a program structure file of the same name is also created. A ".cbl" program file is created the first time you generate the program.
  5. You can add programs to the project with the File/New/Program command. This command creates only a program structure file. A ".cbl" file is created the first time you generate the program.

There is nothing special about creating a project for a new application. For detailed step-by-step instructions, see section 6.3.1.

  1. When you create the project (File/New/Project), use the Standard Project template.
  2. Before adding files to the project, check and adjust the Automatically create a program structure file option in the Tools/Options interface under Environment/Miscellaneous. This option directs the workbench to create, or not create, a program structure file for each COBOL program added. Note that the option settings apply to AcuBench as a whole, therefore it is a good idea to always check the setting of this option before adding files to a project.

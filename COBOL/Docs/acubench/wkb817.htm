8.5 The Control Flow Diagram Tool

A Control Flow Diagram (CFD) is a graphical representation of all possible paths, including evaluation points, branches, and exits, that an execution sequence may take in a program. A CFD employs standard flowchart conventions, using different shapes and colors to represent your program's logic. With this tool, you can check your source code by examining the logic flow indicated in the diagram on the screen or in print.

When you use the Edit/Create/Update CFD command, the Code Editor's CFD tool generates a CFD for the active program in a separate window. You can scroll horizontally and vertically to see different parts of the diagram. This tool also updates the CFD to reflect any subsequent changes to your program.

A CFD may consist of many different types of diagram elements, primarily standard flowcharting symbols. You can identify each element by its name, symbol, and color.

Section and paragraph names appear in Procedure Division sequence order in a drop-down box. You can view a nested diagram for a section or paragraph by choosing its name in this box. The Top push button causes the CFD for the main application logic to display in the window. You can use the Back and Forward push buttons to display the CFDs for other parts of your code.

Note that the CFD utility does not currently support the use of level-78 fields. If your program contains level-78 fields, no CFD is generated.

Generating CFDs for very large programs may use a great deal of memory. When the Code Editor detects that system resources are low, it gives you the option of either continuing with CFD creation or terminating the action.

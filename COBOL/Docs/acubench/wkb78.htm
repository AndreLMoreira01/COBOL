7.3.1 Opening the CFE

The Project/Settings/Runtime dialog provides access to the CFE.

  1. Select Settings from the Project menu or click on the Settings icon. The Project Settings dialog displays.
  2. Click on the Runtime tab. When you display Common Options, the "Alternate configuration file name (-c)" option is located near the top of the Runtime dialog. When "-c" is selected, the runtime uses the specified configuration file. If a configuration file is not specified, none is used.

    3. To edit a remote configuration file, select Thin Client Options from the Catalog drop-down box. This lets you open the remote file in the CFE.

  3. Next to the check box used to set the "-c" flag, enter the path and name of the configuration file. If you specify a local file (in the Common Options section), you can browse to the path and file you want by clicking the ellipsis button. No browse capability is available for remote configuration files (in the Thin Client Options section).

    4. Changes you make are displayed in the "Runtime Options" field, at the bottom of the dialog. It is not possible to edit items in this field directly; the field is updated as you add and remove settings in the various categories of the Runtime tab.

  4. To create a new configuration file, click in the entry field and enter a new file name. The configuration file may reside in any directory. The file's pathname must be valid, however, or an error message displays.
  5. Click the Edit button (to the right of the option) to launch the CFE. The Configuration File Entries dialog opens.

    6. The CFE reads the specified configuration file and builds a list of existing entries that can be accessed from the "Editing entry" drop-down list. The CFE allows you to add or remove variables, change the value of variables, and add comments.

Variables in the file that are commented out (have a "#" character at the beginning of the line) are not recognized, and cannot be uncommented using the CFE.

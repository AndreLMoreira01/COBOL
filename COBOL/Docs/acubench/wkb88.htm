8.1.6 Editing Area

By default, the editing area is divided into five ANSI-specified display fields: Sequence Number area, Indicator area, area A, area B, and Identification area. The size of each field is determined by character position. The total editing area is 80 columns wide. Each field may appear in a different color for easy recognition. The Code Editor offers a wide range of options for customizing the editing area in the Tools/Options dialog box. For example, you can select a different column width and color for each display field. More information about Code Editor customization can be found in
section 4.5, "Setting Code Editor Options."

You should note that the following paragraphs describe each element of the editing area as occupying its default column(s). COBOL functions normally adhere to these ANSI-specified column settings. However, you may want to change the column settings if you are working with terminal source format, which does not use the ANSI default settings. (Terminal source format is described in Book 1 of the ACUCOBOL-GT® manual set.)

Sequence Number Area. The Sequence Number area occupies columns 1-6. Because the compiler ignores this area, its contents can be composed of any characters.

You can place sequence numbers in this area if you want to re-number your text. When you choose the Format/Sequence Number command, the Sequence Number dialog box appears. In this dialog box, you can name the starting sequence number and designate the interval between each number. You can apply your sequence numbering to the entire active Code Editor file or to a selection of lines in your file. If you have selected lines of text in your source code, the "Selection" radio button in the dialog box is automatically selected. You can also use this dialog box to clear the Sequence Number area of all sequence numbers. The Sequence Number command is not available for terminal source format.

Indicator Area. The Indicator area occupies column 7. This area must contain a special character that defines how the compiler treats each line of code. The following table contains the possible special characters:

Character Designation

" "(space)

Normal (default character). The compiler processes the line normally.

"-" (hyphen)

Continuation. This line is compiled as a continuation of the previous line.

"*" (asterisk)

Comment. The compiler ignores the line's contents.

"$" (dollar sign)

Comment. The compiler ignores the line's contents.

"/" (slash)
                             

New page. The compiler responds as to an asterisk, except that this line starts on a new page in the source listing the compiler creates.

"D"

The compiler treats this line as a comment line. However, if the compiler uses the option to include debugging lines, it treats the line normally.

Area A. Area A occupies columns 8-11. This area contains division headers, section headers, paragraph names, and some level indicators.

Area B. Area B occupies columns 12-72. All other COBOL source code text appears in this area.

Identification Area. The Identification area occupies columns 73-80. Any text can be placed here, but the compiler may conditionally compile the contents of this area based on specific character patterns.

You can configure the Code Editor to place a specific text string in this area when a line of code is added or modified. Use the Tools/Options/Code Editor/Format dialog box to accomplish this. Information about this interface can be found in section 4.5.2, "Setting Format Options." A line is considered unchanged if you undo your work before saving or you type some characters to alter your code and then you delete them before saving. Modifications to code entered in columns beyond 80 do not trigger the modify code option.

7.7 Setting Environment Options

AcuBench includes a facility for setting environment variables for the project. You can view, add, modify, or delete environment variables in the Environment tab of the Project Settings dialog.

Five environment variables are predefined by default. They are:

ACUPATH

defines the path to the ACUCOBOL-GT executable files (e.g., c:\Acucorp\Acucbl720\AcuGT\bin)

CODE_PREFIX

defines directories to search when a program name is specified without a full path. See Book 1, Chapter 2 of the ACUCOBOL-GT documentation set.

COMPILERNAME

defines the name of the compiler executable file (by default, "ccbl32.exe")

COPYPATH

defines one or more paths that AcuBench searches to locate COBOL COPY files (see section 7.7.4)

RUNNAME

defines the name of the runtime executable file (by default, "wrun32.exe")

A sixth variable is added when AcuBench verifies that it can connect to a specified remote server and port:

THINNAME          

defines the name of the thin client executable (by default, "acuthin.exe")

As with the RUNNAME executable, the executable defined by THINNAME must be located in ACUPATH.

Note: The value of variables defined in the operating environment, such as those defined in the Environment tab, take precedence over the values of corresponding variables defined in the runtime configuration file ("cblconfi"). For more information, see Chapters 1 and 2 in the ACUCOBOL-GT User's Guide and Appendix H in ACUCOBOL-GT Appendices.

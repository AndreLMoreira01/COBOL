11.7.3 Using the Event Editor

The Event Editor is a graphical interface that allows you to develop code that is directly associated with a control. EXCEPTION, BEFORE, and AFTER procedure code can also be added through the Event Editor. This interface simplifies event coding in that you can code events as you design your screen, consolidating screen design and code generation activities. More information about individual events and event handling can be found in Book 2 of the ACUCOBOL-GT documentation set.

The Event Editor can also be used to control what is automatically generated between the AcuBench "declarative" tags. Data layout names are included in the drop-down "Control" entry field, and Declaratives is an option in the "Message" entry field. When these options are selected, you can code your declarative directly in the Event Editor.

The Event Editor is useful when you need to create code for report generation. Refer to Chapter 14 for information about Report Composer functions and the Event Editor.

If you set the "Jump to Event Editor when browsing" option in the Tools/Options/Environment/Build dialog, you can enter the Event Editor automatically when you click on a compilation error message caused by code in your program's event COPY file. (Refer to section 4.4.4, "Setting Build Options," for more information.) You can also enter the Event Editor in other ways, as described in the following sections.

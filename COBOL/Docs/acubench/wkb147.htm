5. Make one-time modifications to the generated code.

  • Modify the Acu-reportname-MASTER-PRINT-LOOP paragraph in the program's ".cbl" file so that it contains the following code statements:
    • PERFORM BEFORE-MASTER-PRINT-LOOP 
PERFORM UNTIL [EOF-CONDITION] 
    PERFORM Acu-RPT-reportname-DO-PRINT-RTN 
    PERFORM END-MASTER-PRINT-LOOP 
END-PERFORM

    You will note that the PERFORM . . . DO-PRINT-RTN statement is automatically generated outside the AcuBench program tags. Because you enter your master print loop code outside the program tags, it is not overwritten if you regenerate your code after making changes to your report. Make sure that the "Regenerate tagged area only" check box is selected in the Program Properties/Code Generation interface.

  • To view or print your report, add a line of code after the Acu-Initial-Routine paragraph in the source file to engage the generated report code. Be sure to add this code outside the AcuBench program tags so it is not overwritten in subsequent code generations. This code is different for graphical reports and character-based reports. Refer to section 14.2.3, "Using Report Labels and Entry Fields," for a description of the code options for graphical reports. Read section 14.4, "Character-based Sample Reports," to see the code for character-based reports.
  • For instructions on printing a graphical report in Acucorp's Thin Client environment, refer to section 14.3 in this manual.
  • Generated code for handling the report grid control contains additional paragraphs and may require unique modifications. For details, refer to section 14.2.10, "Using the Report Grid Control," and section 14.2.11, "Using Grid TABBODY Paragraph Code with the Report Grid Control."

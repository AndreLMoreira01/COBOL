15.3.5 Report Section Properties

The report Property window displays the properties that you can set to control the behavior and appearance of the sections in your report. The following paragraphs list the properties that apply to report sections. Unless noted otherwise, these properties can be set for each report section described in the previous paragraphs.

Note that a property marked with an asterisk (*) can have its default value set in the Tools/Options/Report Writer/Default dialog. For information about this dialog, refer to section 4.9.2, "Setting Default Options." The section default settings in Tools/Options/Report Writer/Default apply to all report sections. For example, you cannot have different default settings for the report header and the page header sections.

(Name)

The default name is derived from your report name. For example, for Report1, the default report header name would be Report1-RH-1. You can change this name if desired.

Color*

The default value for this property is 131329, foreground black on background white.

Setting the Color property at the section level colors the background of that particular section of the report and sets default colors for all report controls. If you set the Color property for a report control that appears in a section, the control's Color setting takes precedence over the section's Color setting.

Color Variable*

The Value cell for this property has an ellipsis push button that opens the Select Variable interface. If a variable is selected in this dialog, subtle modifications to the generated code occur, such that it references the Color Variable instead of the Color value directly. This property has no initial default value.

Font*

Setting the Font property at the section level sets default fonts for all report controls. A drop-down box displays several choices for font. The default value is "Default Font". The push button in the Value cell causes the Font dialog to appear.

If you set Font for a report control that appears in a section, the control's Font property setting takes precedence over the section's Font setting.

Lines

This property value represents the amount of space on a standard page that is reserved for the printing of a report section.

Note that the standard unit of measure used by Lines is set in the Tools/Options/Report Writer/General interface. In this dialog, you can select Inch (the default) or Centimeter in the Unit frame. The following table shows the Lines property default values for the various report sections:

Print after Page Footer*

This property is available only for the report footer section. It determines whether the report footer prints before or after the page footer on the report. The default value is "FALSE".

Print before Page Header*

This property is available only for the report header section. It determines whether the report header prints before or after the page header on the first page of the report. The default value is "FALSE".

Print Condition

A print condition can be used to make the printing of a particular section conditional. Note that the word "IF" is prepended to your print condition. For example, if you did not wish to print the report header for user "bob", you could create the following condition in that section's Print Condition property: 

USER-NAME NOT = "bob"

You can type the print condition directly into the drop-down entry field area in this property's Value cell (use Ctl+Enter to take the cursor to a new line) or click on the ellipsis push button to open the Expression Builder interface.

Size

This property represents the width of the page in which this section is printing.

Note that the standard unit of measure used by the Size property is set in the Tools/Options/Report Writer/General dialog. In this interface, you can select Inch (the default) or Centimeter in the Unit frame. When Unit is set to Inch, the default value for Size is 8.50 for all report sections. When Unit is set to Centimeter, the default value is 21.59.

Skip Page after Print*

This property is available only for the report header, group header, group footer, and detail sections.

A drop-down box allows the selection of "TRUE" or "FALSE". If you select "TRUE" for a particular report section, a blank page is printed after that report section is printed. The default value is "FALSE".

Skip Page before Print

This property is available only for the group header, group footer, and report footer sections.

A drop-down box allows the selection of "TRUE" or "FALSE". If you select "TRUE" for a particular report section, a blank page is printed before that report section is printed. The default value is "FALSE".

Visible*

A drop-down box allows the selection of "TRUE" or "FALSE". Selecting "FALSE" inhibits the printing of report controls in a section but not the detection of the breakpoint. The default value is "1:TRUE".

Visible Variable*

The Value cell for this property has an ellipsis push button that opens the Select Variable interface. If a variable is selected in this dialog, subtle modifications to the generated code occur, such that it references the Visible Variable instead of the Visible value directly. This property has no initial default value.

Zebra

This property is available only for the detail section.

Setting the Zebra property to a non-zero value between 1 and 16 causes alternate detail section lines to be printed on different background colors. The background color defined by the Zebra property alternates with white. The default value is "0".

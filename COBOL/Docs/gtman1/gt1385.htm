3.7.1 Using the profiler

The profiler is optimized for batch programs, and is especially useful with batch programs that run large numbers of transactions. It is more difficult to get good information from interactive programs. If user wait times are the issue you're trying to solve, trace files are more likely to return useful information than the profiler.

When you prepare to use the profiler, you should make an effort to run your application as cleanly as possible. This means making sure that your system isn't overloaded with large numbers of users, heavy system traffic, and so on. The cleaner the run, the more useful the information returned by the profiler.

The following steps describe how to perform profiling using default profiler and acuprof behavior. Options for configuring profiler and acuprof behavior appear in the next section.

  1. Compile your COBOL programs for debugging.

    2. You must compile with at least the "-Gy" option (to include at least minimal symbol information in the object file) for the profile to contain paragraph information. It is preferable to compile for full symbol information ("-Gs") or for full source debugging ("-Gd").

  2. Execute the program with the "-p" runtime option to create the "acumon.xml" file.
  3. By default, acuprof searches for a file called "acumon.xml" file residing in the same directory from which you execute runcbl. If you use the default settings, this means that you must do one of the following:
    • Copy the "acumon.xml" file to the directory containing the acuprof utility (by default, the "tools" subdirectory in your default ACUCOBOL-GT installation directory).
    • Create a runtime configuration file that adds the "tools" subdirectory to your CODE_PREFIX directories, then execute acuprof from the directory containing your "acumon.xml" file.
  4. Use the command runcbl acuprof to launch the profiling application and process the data from the "acumon.xml" report.

    5. By default, acuprof creates a report file called "acumon.rpt" in the execution directory, then displays a message to indicate that the report was created successfully.

  5. Click OK to end acuprof execution, then open the newly created report file in the text editor of your choice.

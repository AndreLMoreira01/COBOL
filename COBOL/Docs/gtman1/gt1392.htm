3.8.2.1 CHAR-EBCDIC and SIGN-EBCDIC instructions

The CHAR-EBCDIC instruction tells AcuSort to expect data that is encoded in the EBCDIC character set rather than ASCII. SIGN-EBCDIC tells AcuSort that numeric DISPLAY types that include signs should be interpreted according to the EBCDIC convention. The use of CHAR-EBCDIC implies SIGN-EBCDIC. Both instructions must precede all SORT or MERGE instructions.

For example, if your data is EBCDIC, you should use CHAR-EBCDIC. If you have ASCII data with EBCDIC sign encoding, you should use SIGN-EBCDIC. If you have ASCII data with ASCII sign encoding, you would not use either instruction.

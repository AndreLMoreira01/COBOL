4GL_COLUMN_CASE

When set to "unchanged", this variable causes the runtime to leave the case and hyphen usage of the field names found in XFDs unchanged. XFDs are used with the Acu4GL interface, AcuODBC, or AcuXML. They are also required for international character mapping with AcuServer and they provide useful information to the alfred record editor. By default, the runtime converts all field names to lower case and all hyphens are converted to underscores.

For AcuXML, the case and hyphen usage of the XFD must match the XML file exactly, and 4GL_COLUMN_CASE should be set to "unchanged". For Acu4GL, however, you should be aware that most databases do not accept hyphens in column names. If you set this variable to "unchanged" to protect case, you may need to modify the XFD by hand to replace hyphens with underscores.

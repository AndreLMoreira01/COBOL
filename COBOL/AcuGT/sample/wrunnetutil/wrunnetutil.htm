Overview Of WrunNetUtil

WrunNetUtil demonstrates the use of WrunNet and provides a generic
.NET interface to the Acucobol runtime.  This utility can be
used as a .NET testing aid for Acucobol non GUI programs.
When used with an Acucobol GUI program, this utility will end
when the Acucobol GUI program is ended because they both are using
a main window in the same process. 
WrunNet is added to a project as a reference; therefore, all
ecncapsulated methods are exposed allowing the programmer native
.NET interfaces to the Acucobol runtime. 
See the WrunNet specification.

Program Path Top

				
The location of the Acucobol object file. This is the full path
file name. EX: c:\AcuObjects\MyProg.acu 
If you are executing an entry point, other than the
main entry, leave this field blank and enter the information
under Runtime Options preceded by -y.
EX: -y c:\AcuObjects\MyProg.acu.

Entry Name Top

	
If the Acucobol program contains Entry Point Names enter a
name here to start execution at that point. This is used with
the -y option.

Runtime Path Top

Directory path where wrun32.dll, Acucobol runtime, is located.
WrunNet uses the installation registry setting for Runtime Path when
this field is empty.

Parameters Top

To pass parameter data to the Acucobol program press the AddParam button.
A dialog will display showing a parameter number, a combo box populated
with data types and an edit box for the field value.  For each parameter
enter values and select a data type then press the Add button.
When all parameters are entered press the Done button.  As each parameter
is added it will display in the list view on the main screen.
Upon return from execution the Return-Value column in the list view will
display the updated field values.

Global Runtime Options Top

Any runtime options, see Acucobol runtime command line options.
If you enter command line options press the AcuInitialize button before
pressing the Execute button.  Acuinitialize is for global options which
are in effect for the entire life of WrunNetUtil. Only one AcuInitIalize
is allowed per WrunNetUtil execution.

Debug Top

Program instance option Debug invokes the Acucobol Debuger.
See related Acucobol runtime documentation.

Cache Top

Program instance option Cache Sets program caching on.
See related Acucobol runtime documentation.
